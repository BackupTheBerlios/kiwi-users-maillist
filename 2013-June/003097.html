>     I need to make a SLEPOS 11 SP2 image for a 4694-245 IBM POS. This is a
>    very, very old box, sporting an AMD K6-2 processor and 256M of ram.
>     So far I tried to boot the normal Image I use in 4800-722 boxes and it
>    fails with a kernel panic...
>     Is there a known way to builf a kiwi image for smaller/older hardware?
>    Thanks.
>    PS. its this exact
>    crash: [1]http://www.novell.com/support/kb/doc.php?id=7007221

Hmm, I think kiwi can't help much here. So iirc you are facing a problem
due to the i686 vs i585 optimized compile of glibc which has a bad impact
on older cpu's. If you have an SP2 repo without i686 builds it should
solve your problem. Other than that you can try the target-arch option
in kiwi which tells zypper to use only packages of the specified
architecture:

     --target-arch i586

this should also force the use of the i586 compiled glibc and solved
some trouble for me on older 32bit network clients
