In a clean way you would solve that by writing a boot profile but 
I guess you want the fast and dirty solution :-) Here it is:

1) create a repo with your kernel

   mkdir /tmp/mykernel
   cp kernel-2.6.18DMP-1.i386.rpm /tmp/mykernel

2) make your repo available to the kiwi netboot image
   I don't know which distro you want to use for the client so
   the following guesses it is for suse-11.2

   rm -f /usr/share/kiwi/image/netboot/suse-11.2/.checksum.md5
   vi /usr/share/kiwi/image/netboot/suse-11.2/config.xml

   <repository type="rpm-dir" status="fixed">
      <source path="/tmp/mykernel"/>
   </repository>

3) by default the "std" bootkernel profile is used so simply
   exchanging the kernel package inside the std profiled path
   will use your kernel   
   
   vi /usr/share/kiwi/image/netboot/suse-11.2/config.xml
   
   replace:

   <packages type="image" profiles="std">
        <package name="kernel-default"/>
   </packages>

   with:

   <packages type="image" profiles="std">
        <package name="kernel-2.6.18DMP-1"/>
   </packages>

4) last but not least make sure your system image config.xml also
   includes the _same_ !! kernel-2.6.18DMP-1 package otherwise the
   initrd kernel and the system kernel doesn't match

Hope that helps
