On Tue, 23 Mar 2010, Robert Schweikert wrote:

> I saw the patch about the USB boot fix, I think. I am still having
> issues with the feature in qemu for images built with Kiwi 4.23.
>
> As I am ignorant, i.e. I have no clue how this works to begin with,
> about this subject I will ask a question that is probably dumb.
>
> Why can we not get the same implementation that the SUSE install DVDs
> use? The boot from hard drive feature always works even in qemu.

Basically because a DVD is not a disk from the BIOS's perspective. But an
USB stick is. So in the second case you have to do some guessing to find out
how the BIOS disk list would have looked if the USB stick would not be
there.

Ultimately, booting from DVD may also result in a changed disk order, but
this is really rare.

What exactly is the problem?
