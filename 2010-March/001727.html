Hi,

> My name is Lee, and I'm new to the list, so I hope this question hasn't already been answered.
> 
> I just inherited a kiwi build system, and it's working fine, but I'd like to change our Images Grub boot menu timeout to 0, so that the system boots more quickly.
> 
> Is there a way to do this in config.xml, or do I need to write a custom script (perhaps in images.sh)?

You can change that inside the image after it has been built.
This works like the following commands explain:

   losetup /dev/loop0 image.raw
   kpartx -a /dev/loop0
   mount /dev/mapper/loop0p1 /mnt
  
   ===> vi /mnt/boot/grub/menu.lst
   
   change the timeout and store the file

   umount /mnt
   kpartx -d /dev/loop0
   losetup -d /dev/loop0

That's it
