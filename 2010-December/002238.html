Hi,

> 1. Disk must be prepartitioned and formatted to known filesystem, so there
> is enough space for each partition in config.<MAC>

This is weird because before the new partition table is created
kiwi wipes the old one as follows:

  dd if=/dev/zero of=$DISK bs=512 count=1 >/dev/null && \
      parted -s $DISK mklabel msdos

so it really doesn't matter what data the table contains before.
Are you using the latest version of kiwi ? parted is part of the
11.2 netboot code there

> 2. No "x" in size field of config.<MAC>: only precise numbers that are
> slightly more than enough to contain the image.

'x' means use the entire rest space available, so it must be set
only once and that worked for me too... strange

> 3. Mount point should exist (just mkdir /tmp/mypxe/mountpoint after
> "prepare" kiwi step but befote "create" kiwi step)

this is correct yes
