I used the latest kiwi version to reproduce this and it worked
fine on my test system. I changed the config.<MAC> as follows:

  DISK=/dev/sda
  PART=5;S;x,251;83;/,x;83;x,
  IMAGE=/dev/sda2;LimeJeOS-openSUSE-11.3.x86_64;1.11.3;192.168.100.2;4096
  UNIONFS_CONFIG=/dev/sda3,/dev/sda2,clicfs
  PARTITIONER=parted

I checked the created md5 files as follows:

  cat LimeJeOS-openSUSE-11.3.x86_64-1.11.3.md5

  65a6c76e45b2489bdfa91d74145773b5   4153 28977

  md5sum LimeJeOS-openSUSE-11.3.x86_64-1.11.3.clicfs

  65a6c76e45b2489bdfa91d74145773b5  LimeJeOS-openSUSE-11.3.x86_64-1.11.3.clicfs

which kiwi version did you use ? Your checks are correct the md5
sum must match. You can't check the md5sum with the '-c' option
though because for kiwi I additionally add the blocks and block-size
information needed for atftp download so it's not an original .md5
file suitable for -c anymore
