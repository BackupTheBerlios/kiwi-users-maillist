On 12/10/2010 01:37 PM, Kalman Szabo wrote:
> Dear MArcus and other kiwi-users,
> 
> what is the canonical way of adding kernel-modules/libraries/binaries to
> the image, which are not packed into RPM's?

You have two oprions:

1.) Create a tarball, place the tarbal at the same level as your
config.xml file in your configuration tree and include the tarball in
config.xml via <archive name="...."/>

The files in the tarball need to be in their proper location starting
from the root tree, i.e. opt/myApp/bin/myExec in the tarball will end up
as /opt/myApp/bin/myExec in the final image.

2.) Place all your file in the overlay tree (i.e. inside the directory
named "root") inside your configuration tree. No modifications need to
be made to config.xml in this case.
