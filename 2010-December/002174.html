> How can we access information obtained during preHWdetect.sh
> from the preCallInit script when stored to a temp filesystem?

It should be possible to store the data in

   /dev/shm/...

in preHWdetect.sh. Later in preCallInit you should still have
the possibility to access the data in /dev/shm. I use this to transfer
the log file from the initrd into the later system so it should
work for other data too.
