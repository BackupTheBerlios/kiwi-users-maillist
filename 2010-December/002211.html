On 12/13/2010 09:48 AM, Till Dörges wrote:
> Hello everybody,
> 
> I was trying to pass a parameter to the ramdisk module (brd.ko). Since this is loaded
> pretty early by kiwi, one apparently cannot simply modify the root fs of the
> generated kiwi image and add a modprobe.conf file or similar.
> 
> I'm using a USB boot image. So I was wondering whether it would be possible to add a
> conf file for modprobe into the *boot* image (where linuxrc resides). I tried with
> .../usbboot/suse-11.3/root/etc/modprobe.d/99-local.conf but with no effect (the very
> same file works on my desktop system).

To get something like this into the initrd you'd have to create a
tarball, then include the tarbal in kiwi via the <archive> element and
set the bootinclude attribute to "true". The file in the tarball must
have the proper location in the root tree, i.e. if I wanted something in
/opt/myApp then inside the tarball the path would be

opt/myApp/myFile.ext

> 
> I managed manually by adding a kernel boot parameter (rd.rd_size=<size>).
> 
> However, I did search the documentation and couldn't find a hint how to acquaint KIWI
> with my kernel boot parameter.

Use the kernelcmdline attribute of the <type> element to set any kernel
command line parameters.

> I'd greatly appreciate a pointer to do an RTFM.

You can find this in the XML documentation

/usr/share/doc/packages/kiwi/schema/kiwi.html
> 
> 
> While at it, I'm wondering whether modifying the boot image with a modprobe config
> file shouldn't work, too.

It would, see above.
