I have an openSUSE 11.2 KIWI image that is booted on a diskless system
via dhcp/pxe. The image takes on various personalities based on a
configuration option in dhcpd.conf on the boot server. The value of the
option is happily made available as an environment variable during boot,
and is acted upon by a init.d script in the boot image. In fact, I have
been using the 'option domain-name' parameter to pass along this
information because I do not otherwise use it. I will be changing the
make-up of the network, and using this option this way will become
problematic.

I really like specifying this in the dhcp configuration as it results in
a one-stop-configuration solution. So I am looking for a way to
accomplish the same thing, but without using such an important variable.
Nonetheless, I thought I would check if anyone has a suggestion of a
better way to do this sort of thing. All suggestions welcome!
