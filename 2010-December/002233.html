On 12/24/2010 12:17 PM, Mike Turchenkov wrote:
> I know about, e.g.,  <package name="vim" bootinclude="true"/>  in
> config.xml but when I get emergency console after some errors during
> pxeboot, it doesn't know "vim"

You are falling victim to the initrd clean up algorithm. This was fixed
last week but packages with the fix are not yet available.

As a work around you can put everything you need to run vim into a
tarball that extracts at the root level, for example the tarball
directory could be like this:

vim/bin/vim
vim/etc/vimrc
...

then include the taball via <archive name="..." bootinclude="true"/>
