Hi,

> > Thank you for the response, Carlos. If I add the pattern, should I
> > leave package configuration as is as shown ?
> 
> I think you can just use patterns instead of packages and it will work. Some 
> recommended packages might not be included but for sure you will notice if 
> that happens.

I agree, it has turned out that the best way is to use a pattern
(if it exists for the purpose you need the image for) and add those
packages which are the "special" ones you additionally need and are
not installed by the package manager due to the dependencies of these
packages. This keeps the list small and maintainable

> > Another question, How if I want to know another pattern provided by
> > kiwi ?

kiwi does not provide patterns, patterns are provided by the repository
and the package manager which handles the repository should have the
functionality to display what patterns are available. As the patterns 
we talk about are something SUSE specific smart can't handle them but
zypper knows about it:

	zypper patterns

> Is this same pattern with yast repositories pattern ? 

yes

> Is there any special command to display available pattern ?

not in kiwi

> I don't know which the best way to list them is, but I use patterns-openSUSE 
> (SRPM). It is great to check which packages a pattern require, recommend and 
> suggest. You can also use 'zypper se | grep pattern'.

yes you should use the package manager functionality to show these
kind of information.
