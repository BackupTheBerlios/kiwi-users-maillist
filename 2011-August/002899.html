> So it's
> config.xml
> root/testfile

> config-cdroot.tgz contains a file named "roottestfile"
> 
> I build the iso with
> kiwi --prepare configdir --root rootimage
> kiwi --create rootimage --type iso -d isodir
> 
> During iso creation i see that the tgz file is copied over to iso
> directory white squashfs image is being created, although not
> unpacked. And it doesn't seem to end up in the final iso image, nor
> its contents.

The meaning of config-cdroot.tgz is that the contents of this archive
are packed as files to the iso filesystem. So you should be able to
do this:

   mount -o loop image.iso /mnt
   ls -l /mnt

   ==> see your "roottestfile" in that listing ?

If you can't see that file it's a bug. If you expect roottestfile
to be part of the image itself it's a misunderstanding of the
meaning of config-cdroot.tgz. Normally people use config-cdroot.tgz
to store a license file or some README or help information as part
of the iso

Could you check this ?
