> However, I've had some issues, too.
> I'm trying to build a system with kde4 using opensuse11.1(betaXX)
> 
> I was using this:
> 
>        <packages type="image">
>                <package name="kernel-default"/>
>                <package name="bootsplash"/>
>                <package name="bootsplash-branding-openSUSE"/>
>                <package name="vim"/>
>                <opensusePattern name="default"/>
>                <opensusePattern name="kde4"/>
>        </packages>
> 
> but it did not include kdm. Even so, when I tried to log in (using
> xdm) I got an error about dbus.

yep that's because no recommended packages have been installed
the way you setup your packages section. Just include the following
and it should work:

    <packages type="image" patternType="plusRecommended">
        <package name="kernel-default"/>
        <package name="bootsplash"/>
        <package name="bootsplash-branding-openSUSE"/>
        <package name="vim"/>
        <opensusePattern name="base"/>
        <opensusePattern name="kde4"/>
    </packages>

> however, why weren't they included if they were necessary for
> operation? Is this a patterns problem wherein the pattern for kde4 is
> missing the prequirement?

inside of the patterns you have a distinction between required
recommended and suggested. if you don't specify any level only
the required packages will be installed but this does not mean
the system has everything to run smooth in all areas. Especially
on the desktop side you really need the recommended packages too.

It sounds odd, yes it does but that's how the policy applies for
the opensuse patterns which I can't change. I did not make recommended
to be the default to allow small images based on patterns too
but as result of that people suffering the consequences of
setting a patternType
