Marcus Schäfer wrote:
> Hi,
> 
>>  From the documentation it is not clear to me what the difference is 
>> between "patternType" and "patternPackageType".
>> A more detailed explanation would be appreciated.
> 
> kiwi has its own pattern solver which is used if another
> packagemanager than zypper is used and no satsolver binding
> is available. Only in that case both attributes patternType
> and patternPackageType needs to be specified. In any other
> case only patternType is evaluated.
> 
> SuSE pattern files (.pat) as you can find them in for example
> a SLE10 repo: suse/setup/descr/*.pat distinguishes between
> req (required), rec (recommended) and sug (suggested) packages
> and patterns.
> 
> * The patternType defines which patterns of the types above
>   should be included
> 
> * The patternPackageType defines which packages of the types
>   above listed in the patterns should be included.
> 
> Example:
> 
> pattern a
>   req: pattern b
>   req: pattern c
>   sug: pattern d
> 
> pattern b
>   req: package x
>   sug: package y
> 
> patternType="onlyRequired" patternPackageType="plusSuggested"
> would include the patterns a b and c and the packages x and y
> 
> This relatively old code path in kiwi is not used very
> often as the pattern solving is mostly done by zypper directly
> or with help from the satsolver binding when using another
> package manager
> 
> Would you mind to add that information to the docs in surely
> better English than I can express it ? :)
