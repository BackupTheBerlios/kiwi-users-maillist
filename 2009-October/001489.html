Hello
> Hi,
>
>   
>> I changed the way kiwi sets up the root device in grub and the fstab
>> (generated by KIWILinuxRC.sh). By default the udev mechanism is used to
>> determine the devices (/dev/disk/by-id/*). I have a special use case where a
>> disk sometimes will be directly connected by pata and sometimes through usb.
>> The standard kiwi mechanism will not work because the id of the disk will
>> change. I therefore patched   KIWILinuxRC.sh to use the uuid of the partition
>> which will not change when the disk is connected differently. 
>>
>> Is there interest to include it in the standard  KIWILinuxRC.sh?
>>     
>
> This is interesting I also thought of changing from by-id to by-uuid.
> I'm not completely convinced what the better method is. But yes if
> you are willing to share a patch with us, we would welcome it :)
>   
There are also certain use cases where "LABEL=" is preferrable over
UUID, since UUID will change if a partition is recreated.
So maybe it should become a configurable option if ID, DEV, UUID or
LABEL are used for fstab/grub?

Bernd
