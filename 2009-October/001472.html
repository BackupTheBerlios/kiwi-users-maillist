Hi,

> I have trouble creating an SLES10 SP2 x86_64 image on
> a openSUSE 11.0 (i586) platform. How can I
> specify the target architecture? Using a 32-bit
> SLES10 repository prep runs without any error, using
> a x86_64 repository the packages are not found.
> What am I missing?

you can't create a 64bit image on a 32bit machine. The
underlaying architecture must match the imaging architecture
(except for x86_64 which also allows to create 32bit images)
The binary execution format has to be compatible with the
machine which is not the case when you try to run 64bit software
on a 32bit machine. The packagemanager detects that and does
not allow to install such software

* There are tricks to emulate that and preload some libraries
  to allow the installation of foreign arch packages. As long
  as they are not executed it shouldn't be a problem but the
  big picture introduces all kinds of other problems when doing
  this (example: chroot into the new root tree is not possible then)

* There are also emulators (Qemu) which allows emulation of a 64
  bit platform on a 32bit machine. But all that slows down the
  image build process to an unacceptable situation.

So the best solution is to build 32bit and 64bit x86 based images
on a x86_64 build machine
