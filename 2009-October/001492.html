This does not sound like a kiwi problem but a problem with the
DHCP server. The kiwi code which tries to find the TFTP server works
like this:

1) it checks if there is a host tftp.$DNSDOMAIN
2) if that doesn't exists it assumes the DHCP server is also
   the TFTP server and tries to assign $DHCPSIADDR or $DHCPSID

all those tests failed in your network. Therefore I assume your
DHCP server is misconfigured. The DHCP client creates a file with
information provided by the server this file is in

   /var/lib/dhcpcd/dhcpcd-*.info

you can pxe boot with the kernel option 

   kiwidebug=1

and check in the shell what contents are in that file
Additionally the /var/log/boot.kiwi file tells use more as
cyberorg suggested
