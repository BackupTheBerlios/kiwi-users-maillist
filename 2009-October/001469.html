Hi,

> Is there any way to prevent kiwi from using aufs?  At the moment we
> are using a kernel that has no aufs module available for it, and as
> such we can't build any images using kiwi.
> 
> From our build log it looks like aufs is used to mount the
> repositories, but i have tried to make sure we use clicfs.  Our aim is
> to provide a hybrid ISO similar to that of Factory/11.2 but the
> requirement for aufs is a blocker.

aufs is not used in 11.2 anymore and kiwi already has no reference
to aufs in 11.2 anymore. I'm wondering where you saw any aufs mount
with the specs I see below:

> Our kiwi file has the following:
> 
>  <preferences>
>     <!-- format can be iso for burning to DVD -->
>     <type
>       primary="true"
>       boot="isoboot/suse-11.2"
>       bootprofile="default"
>       bootkernel="std"
>       flags="clic"
>       hybrid="true"
>       checkprebuilt='true'

maybe you use an old prebuilt boot image. I suggest to remove
the line checkprebuilt="true" and recreate the image
