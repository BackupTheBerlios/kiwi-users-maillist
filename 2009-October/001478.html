Hi,

> When the reboot feature is enabled after dumping an OEM image to a 
> machine's hard drive the media is ejected. This is a very nice feature. 
> However, there are a couple of potential problems with this. If the tray 
> is blocked (some servers hide their drives behind doors) or if the 
> CD/DVD is not removed the boot CD/DVD with the OEM image ends up back in 
> the machine and on reboot the machine ends up booting from CD/DVD again.
> 
> Any chance a confirmation dialog can be inserted at the point of reboot? 
> Something like this maybe
> 
> System reboot required, please remove the media from the drive

I added a confirmation dialog which asks the user to remove the
install media. I also removed the automatic eject due to that change.
I thought it would be a nice thing but you are right there are
problems with it
