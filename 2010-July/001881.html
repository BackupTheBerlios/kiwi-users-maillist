Changes Report kiwi v4.44 2010-06-27 - 2010-07-02
--------------------------------------------------
most important changes for this weeks release...

Special thanks to Robert who spent some time to improve
the code quality and sent many good patches. Your work
is really very much appreciated

==

GIT: fixed repartition problem with new and old parted. The
     code has been changed in a way that it uses a sector
     based start value and a cylinder based end value

when creating images with new parted v2.2 the first partition
starts at sector 2048 whereas an older version of parted (1.8)
uses sector 63 for the start the first partition. This causes
problems when re-partitioning a disk created by another
parted version compared to the one which does the actual
repartition of the disk. SUSE 11.3 will come with new parted
and therefore we had to find a fix for this problem.


GIT: added volume label for recovery partition (bnc #618276)

if you use oem-recovery in your oem image it's now possible
to mount the recovery partition by the label named "recovery"

  mount -L recovery /mount/point

GIT: added xslt stylesheet to convert from schema v4.6 to v4.7
     the conversion moves the vmwareconfig and xenconfig sections
     into the common machine sections and also changes
     format="iso" into installiso="true" and format="usb" into
     installstick="true"

the git comment says all I guess :-)

GIT: Cleanup formats in kiwi:
      * create new KIWIImageFormat.pm module for the conversion
        into different image formats like vmdk,ovf and friends.
        Also added function for creating the machine configuration
        into this new module
      * create common XML specification for vmware|xen config
        starting with <machine>
      * added new option --convert to kiwi.
      * recursively call kiwi with $main::Convert set to convert
        into the specified format and/or configuration
      * added bool attributes installcd and installstick replacing
        the old format="iso" and format="usb" setup

There were some confusion about the image formats in kiwi.
Basically an image format is the format of a disk image. We
currently support raw,vmdk,ovf,qcow2. raw is always the base
for a disk format and therefore not an option. vmdk,ovf and qcow2
can now be specified as format="..." in the <type> section.
The confusion about format="iso" and format="usb" which are
creating an additional installation media for the image has
been solved by the new attributes installiso="true" and
installstick="true"

Have fun and a great (rest)weekend :-)
