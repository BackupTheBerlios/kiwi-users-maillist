Hi,

> I'm trying to create a LiveCD based off of SLES 11 SP1 JeOS, but I
> only want the CD to serve as an installation medium -- not as a trial
> version of the OS. As I'm required to have this CD work with arbitrary
> hardware as well as be the source for a VM installation, the VM
> installer and OEM image installer won't work -- it's LiveCD or nothing
> as far as I understand the Kiwi documentation.

The image type you need here is an oem plus install media like this

<type image="oem" 
      filesystem="ext3"
      boot="oemboot/suse-SLES11"
      installiso="true"
/>

The image and the install iso works on arbitrary as well as a
installation media for VM systems

> Basically, what I want is something that works like a run-of-the-mill
> SuSE installation CD without the option to run a live desktop. Here's
> what I imagine the process should look like once the user inserts the
> CD and boots off of it:
> 
> 1) A graphical menu appears at or just after initial boot time with
> "graphical install" and "text install" options on it, but no option to
> actually start the desktop environment/etc. off the CD. It's okay for
> the CD to display a text-only menu and only have the "text install"
> option if it is running on hardware that is not graphics-capable.

The install iso created by kiwi drives the installation procedure
in a text based environment. Dialogs are provided to you by the
"dialog" program. So there is no graphical installer for this task.
 
> 2) As soon as the user selects one of the install options, it will
> then finish booting the OS (if necessary) and run through the
> installation procedure, which will have some custom AutoYaST
> configurations and .yac modules, as necessary, to automate parts of
> the installation the user is not to change while still asking the user
> for things such as network configuration.

All this can be achieved by a an autoyast profile where you can
do everything except for installing packages because all this is
encapsulated in the image itself. For more information about how
to setup autoyast please contact Uwe Gansert CC'ed

> 3) The final installation on the device's hard drive will only contain
> what's necessary to run a text-mode only appliance. Even if X11 was on
> the CD to facilitate a graphical installation, it will not be
> installed on the destination appliance.

you can influence all that yourself by defining what pattern
and packages should make up the oem system image

> Can anyone provide me with any tips or even HOWTOs/websites on how to
> accomplish my task? Thanks!

you can start with the JeOS templates provided by kiwi-templates
package or use one of the oem examples as they are provided by
the kiwi-doc package
