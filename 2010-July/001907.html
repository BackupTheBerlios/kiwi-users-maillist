Hi Marcus,

Thanks for the response. Unfortunately the terminals are not grouped by
IP, but by function. In addition the MAC addresses are not group-able
via the standard PXE abilities due to a mix of hardware vendors being used.

What would the feasibility of incorporating a means of being able to
create a groups file that would act as a third layer on top of the PXE
structure?

We are attempting to transition off of Thinstation as that project has
had little to no activity over the last few years, and we really need to
be able to update/upgrade images as new applications come online. One of
the most useful facilities we use extensively that Thinstation provides
is the groups capabilities and environment variable loading facilities
to "tweak" each station group.

> no .MAC config found it searches for a hex representation of the
> ip address. So you can group by the four tuples of the ip address.

> function in /usr/share/kiwi/modules/KIWILinuxRC.sh
>
> At the end of that search there is the config.default which is
> loaded. So if most of your machines have the same setup you can
> use config.default and maintain a "black" list
