Changes Report kiwi v4.45 2010-07-04 - 2010-07-09
--------------------------------------------------
most important changes for this weeks release...

GIT:  fixed convert38to39.xsl stylesheet

 if you use XML descriptions at version 3.8 or earlier the
 automatic conversion via xslt will fail for the bool oem
 parameters with value yes|no
 

GIT:  moved in-place recovery file creation into suse-dump
      directly after the install verification (bnc #620777)

 oem-inplace-recovery is a method which allows the creation
 of the recovery data on first deployment and thus saves space
 inside the image where it is stored in default mode. The
 system now will create the archive inside the install code
 and if that is not possible (or not called) on first boot
 directly after the repartitioning step


GIT:  fixed build for s390

 This is the last commit in a series of changes to support
 s390 imaging. The s390 support in kiwi requires the KVM
 virtualization layer. given that this is installed on the s390
 machine one can create vmx and oem images with kiwi for s390.
 It's a major milestone for the kiwi project to support this
 architecture. Building images with kiwi on s390 is fun because
 the I/O performance on this systems is fantastic. This version
 of kiwi also provides JeOS templates as example image descriptions
 so that it is easy to create images on s390. The target customer
 for s390 are always enterprise customers, therefore the target
 distribution for imaging is SLES. Big thanks to Alexander Graf
 who helped me a lot to understand more about the mainframe
 world and also how kvm works on s390 along with the virtio
 block layer

Have a great weekend :-)
