Hi,

> Something that has occurred to me, that I can't find any details on in
> the documentation, is that I really don't want to have to manually
> create 400+ config.xx:xx:xx:xx:xx:xx files when I finally start to
> deploy this image, and not all of the systems are doing the same thing,
> so I really need to create 10-15 "groups" where these systems will be
> linked into.
> 
> Other then maintaining 400+ symlinks to simulate groups, is there a
> defined/proper way to group like systems into a common config structure?

kiwi supports the same search algorithm than pxe does. If there is
no .MAC config found it searches for a hex representation of the
ip address. So you can group by the four tuples of the ip address.
Would that work for you ? See the searchAlternativeConfig()
function in /usr/share/kiwi/modules/KIWILinuxRC.sh

At the end of that search there is the config.default which is
loaded. So if most of your machines have the same setup you can
use config.default and maintain a "black" list

Another grouping is not possible at the moment so you need to
live with links to the MAC in that case. I'm sorry
