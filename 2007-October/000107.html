Hi,

> I'm using ext3 and this is the line from the exportfs file:
> /live/root *(rw,no_root_squash,sync,no_subtree_check)

that looks good. If you mount that path manually, let's say from
your workstation:

	mount -t nfs IP:/live/root /mnt

does the mount happen rw ? could you store data in /mnt ?
If yes your export is fine. the kiwi linuxrc simply mounts the
filesystem like this.

> Can this have something to do with the suseActivateDefaultServices?

no, *imho*

> I think that the reason for this is that the system is not shuting
> down properly and hence leaving something left in the root tree.
> Is there any file lock set or something like this?

if you mount the root of a linux system via nfs the root stays busy
and a "normal" shutdown sequence can't umount the root filesystem.
That's ok because to be able to call the umount command you need
access to your root dir and in order to shutdown the network
service needs to disabled. So both conflict and rebooting a network
client should happen without the network service to be disabled
by a shutdown sequence.
