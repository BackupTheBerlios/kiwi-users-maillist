Hi,

> I'm attempting to create a SLED10 LiveCD using Kiwi, but I am having a
> problem.
> 
> o  Installed openSUSE 10.3 
> o  Copied contents of DVD to /mounts/SLED10
> o  Created config.xml and build other environment
>  - Typed
>    <type primary="true" boot="isoboot/suse-SLED10-SP1"
> flags="unified">iso</type>

flags="unified" not a good idea for SLE products, they don't provide
aufs/unionfs and squashfs. Just remove the flags attribute

>  - Typed
>     <source path="/mounts/SLED10"/>
> o  Run kiwi --prepare /usr/share/kiwi/image/SLED10-thin --root
> /root/mkdvd
>       this runs successfully

ok

> o  Run kiwi --create /root/mkdvd -d /root
>      Error message is displayed:
> Nov-21 15:20 <1> : Copying boot image and kernel [i386]cp: target
> `/tmp/kiwi-cdboot.E14559/kiwi-cdboot-14498/CD/boot/i386/loader/linux' is
> not a directory
> 
> What kind of mistake do I do? Any help would be greatly appreciated.

Hmm, that could happen if the glob match:

   /root/initrd-isoboot-suse-SLED10*i686*.kernel

matches multiple files which shouldn't happen. Maybe the environment
of /root is not clean or there is another kiwi process running.
