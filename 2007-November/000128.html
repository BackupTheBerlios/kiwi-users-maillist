Hi,

> I've got a Via-based computer PXEbooting, pulling down the netboot
> image, running linuxrc, and having trouble with the IDE device.  The
> via82cxxx, ide-core, and generic modules get loaded, but the device
> nodes never get created (/dev/hda, /dev/hda1, /dev/hda2, etc...).
> 
> The linuxrc script fails when trying to verify the partition table, so I
> exit to a shell (very nice!!!).  Since there are no device nodes, the
> following linuxrc command actually just creates a file:
> dd if=/dev/zero of=/dev/hda bs=512 count=1
> 
> I can manually create the device nodes and partition the disk, then exit
> the shell and the image load continues normally.
> 
> How can I figure out why those device nodes are not being created???  I
> tried running udevtrigger with udev logging on "debug", but no luck
> there.  Any thoughts?

* normally the nodes should appear as /dev/sdX because they should
  be handled by the ata layer

* if it doesn't work you can try to enter the following at the
  boot prompt:

	kiwikernelmodule=ide-disk

  that's the old ide driver and the disk should appear as /dev/hdX
