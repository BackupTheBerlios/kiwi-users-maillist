Hi,

> > Right now the compressed image on an oem ISO only uses the image
> > name from the config.xml file, and does not include any version
> > information.  We would like to be able to tell the version of
> > the ISO image from the compressed image file name.
> > 
> > Can the filename of the compressed image on an oem ISO include
> > the version information?
> > Or is there any way we can set the image name from the kiwi command line?
> > 
> > Right now we are using sed to change the name in the image settings (in config.xml) before running our kiwi build so that the includes the version information.
> 
> Hmm, would it help if I place a file in the installation iso so that
> you can easily check the version information ? If that's ok for you
> what do you prefer, the version as part of the filename or a file containing
> the version information and maybe more ?

Alternatively you can change your config.xml to add a volume ID

  <type ... volid="version-string"/>

you have room for 32 characters here and they will be placed directly
into the iso. You can then check the volid with the following
command:

  isoinfo -d -i /tmp/mytest/LimeJeOS-openSUSE-11.3.i686-1.11.3.iso

  ...
  Volume id: KIWI CD/DVD Installation

as you can see by default this is set to the value above but you
are free of choice here. That also has the adavantage that you don't
have to mount the iso just to grep the version information
