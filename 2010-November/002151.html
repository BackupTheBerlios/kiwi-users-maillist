Hi,

> I am wondering what is the best approach to run kernel related
> commands e.g. depmod which depend on the stuff like uname (2).
> 
> In this case it happens that the configuration of the host kernel
> is taken into consideration by default instead of the target kernel
> configuration.

I strongly recommend to run such commands only when the desired kernel
is up and running. If you build an image with kiwi it creates a new
root tree below a selected path. You can chroot into that tree but you
can't change the running kernel. So if the image you are building uses
the exact same kernel as the system which currently runs you can use
commands processing kernel information. But in most cases this is not
the case and the active kernel differs from the one in the chroot.

Therefore if you have the need to run such commands you should consider
to move them into a boot script boot.local and friends and do it when
the system boots up.
