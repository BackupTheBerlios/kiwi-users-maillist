On 11/16/2010 04:45 PM, john.lund at emc.com wrote:
> Hello Marcus,
> 
> I have a request from my team to update the oem-unattended mode so it does not ask the user which device to install to when multiple storage devices are found.  We have a suggested change to the "dump" script that will do this:
> 
> --- /usr/share/kiwi/image/oemboot/suse-SLES11/root/dump 2010-09-16 13:50:53.000000000 +0000
> +++ ../oemboot/suse-SLES11/root/dump    2010-11-03 16:07:29.000000000 +0000
> @@ -207,6 +220,11 @@
>                 # Found one single disk... use it
>                 #--------------------------------------
>                 instDisk=${deviceArray[0]}
> +       elif [ ! -z "$kiwi_oemunattended" ];then
> +               #======================================
> +               # Unattended mode... use first disk
> +               #--------------------------------------
> +               instDisk=${deviceArray[0]}
>         else
>                 #======================================
>                 # Found multiple disks...

I think this makes sense. We should update the documentation along with
this change to reflect that unattended mode always installs onto the
first device found. THen again I just noticed that oem-unattended is not
documented at the moment. :(

The attached patch rectifies that situation. (Already included the
wording for the new behavior.)  ;)
