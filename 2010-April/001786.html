Changes Report v4.31 2010-04-25 - 2010-04-26
--------------------------------------------------
The most important changes to mention for the last week...

GIT: added alarm timer for HTTP requests. If a URL can't be
     resolved due to problems on the remote side the request
     will be canceled after 5 seconds

  If you are using remote repos and the remote server is reachable
  but doesn't respond to e.g HTTP requests it makes sense to stop
  waiting for it after some time 

GIT: use -t instead of -T for mke2fs
 
  This small change has a big impact on the size calculation
  of the image file. If you ever run into the "no space left on device"
  problem make sure this fix is part of your kiwi installation.
  For details on the difference between -t and -T take a look
  into man mke2fs

GIT: added support for OEM partition based install (bnc #597679)

  The standard OEM installation media (iso or usb) stick created
  by kiwi will wipe the entire disk because the installation process
  simply dumps the virtual disk image on the real disk device and
  adapts it to the real geometry on first deployment. There are
  more and more people who already have a system installed and
  wants an appliance but without loosing their installed system.
  With this feature kiwi will install the relevant parts of the
  virtual disk image into existing empty partitions on the real disk.
  The user has to make sure these partitions exists before, otherwise
  kiwi stopps the installation process and prints a message describing
  which partitions and partition-types it needs to install the appliance.
  I also added a section describing this feature to the OEM chapter
  of the kiwi cookbook (kiwi-doc package). In order to use it
  your config.xml file needs to include the following information

  <type ...>
     <oemconfig>
        ...
        <oem-partition-install>true</oem-partition-install>
     </oemconfig>
  </type>

Have fun :-)
