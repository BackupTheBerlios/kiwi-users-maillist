Hi,

> > > With the next release of kiwi (4.32) I plan to remove the
> > > kiwi pattern module (KIWIPattern.pm). This module was used to solve
> > > the contents of the pattern files provided with the SuSE distribution.

To clarify it more detailed. If we use zypper, zypper does the
pattern solving for us. If we use another packagemanager (smart)
the kiwi satsolver module does the pattern solving for us and only
if we don't have libsatsolver and don't use zypper, kiwi's old
pattern module is used to parse the pattern files.

The support for kiwi starts with 10.1 if you use an older distro
I can't guarantee that something works and we also won't fix it
So there is only a very small time slot between 10.1 and 10.2
where people doesn't have access to libsatsolver because it doesn't
exist and they had problems using zypper together with kiwi which
was the reason why they sometimes choose smart. Beginning with 10.2
there was libsatsolver and also the SLES10 (base 10.1) service
packs provided zypper versions which where usable with kiwi
directly.

The pattern solving code was done to allow those people who use
smart and can't use libsatsolver to use the patterns.
But from a today's perspective I think this code is no longer required.
The module additionally will not work on newer (satsolver based)
distributions because the metadata exists in solv files or repodata
information only which are not handled by the KIWIPattern.pm module
which can only parse that strange .pat format. In fact it's really
more or less useless code which I'd like to remove
