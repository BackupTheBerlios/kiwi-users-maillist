version from here:

Also make sure you have installed the package kiwi-templates

> During the create-phase (I think), kiwi makes a 1GB container, which
> holds a compressed root partition, a small boot-region and a 300MB
> writeable partition. 
> However, if I use a 2GB, 4GB, or even 8GB stick it looks like just 1GB
> is used. Where is this size defined, so i can adjust the available size?

The suse-live-stick example for 11.2 is based on the clicfs compressed
filesystem. clicfs is a copy-on-write filesystem which means all write
actions happens into another device. This device can be either RAM space
or a persistent location.

* If you have built the image type "oem" from your live-stick-example
  kiwi expands the partition it has created for the write actions to the
  available real address space of the disk on first boot.

* If you have built the image type "vmx" from your live-stick-example
  kiwi just uses the virtual disk as it is. Even if you dump that image
  on a USB stick it will not use the real gemotry (vmx is normally meant
  for full virtual systems)
