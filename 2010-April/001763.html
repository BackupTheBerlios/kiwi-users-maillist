I have a condition I cannot solve. I have a set of 8 identical diskless
computers. I have been using an older (> 1 year) openSUSE 10.3 based
KIWI diskless image with them, and they work as expected.  I am now
trying to deploy a more recent KIWI diskless image based on openSUSE
11.2. Oddly, one PC fails during the boot in a consistent way. And only
with these new 11.2 images. 

The systems obtain their initial image via tftp. In the system that
fails, it gets to where it is printing this:

md: autorun ...
md: autorun ... DONE
VFS: Cannot open root device "<NULL>" or unknown-block (3,1)
Please append a correct "root="

Then a kernel panic

I do not have per-system configuration. The only config difference is
where DHCP assigns the IP address based on the MAC address. And that
seems to be working. For everything else, the systems share a common
diskless image and all config files. 

After the system fails, I can still reboot other systems. So the needed
components on the server must be there.

I am guessing it is a hardware issue. But the odd thing is that it only
occurs with the new 11.2 images. The older 10.3 images still work for
all systems...

Any suggestions?
