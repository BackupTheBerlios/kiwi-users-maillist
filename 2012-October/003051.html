I guess with 'new version of the ISO' you mean an install ISO which
installs the image on the device at boot time. I also guess the
device has some local storage e.g a disk or an SSD or something
which the install ISO has installed the image to

If this applies it would be possible to upgrade the local storage
on the fly with the following process:
 
  - download the new ISO
  - loop mount the ISO
  - loop mount the .squashfs file in the ISO
  - dd the .raw file to the local storage
  - reboot

BUT ! this will not preserve any custom data written to
the local storage since the last time the machine was
installed
