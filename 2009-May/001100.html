Hi,

> i found in one tutorial the option to override the created isolinux.cfg
> by placing it in /cdboot/isolinux.cfg in the config.sh path.
> But as you can imagine this file is ignored.
> 
> 1. Why can´t the generation of /cdboot/isolinux.cfg not be part of the 
> --prepare process? (dream)
> 2. Where can i actually place a further kernel option like vga=0xXYZ?
>     Because naive changes in KIWILinuxRC.sh of DEFAULT_VGA lead to 
> nothing.

It works the way you dreamed of :-) meaning isolinux.cfg is
created as part of the create step. It's the create step because
kiwi is able to create different image output formats from the
same prepared tree so the differentation between the image
types and boot methods happen when creating them

See KIWIImage line 1842

    #==========================================
    # setup isolinux.cfg file
    #------------------------------------------
    $kiwi -> info ("Creating isolinux configuration...");
    ...

If you have a kiwi version running using the old style
I strongly recommend to the latest version
