Hi,

> I wrote the user specification listed below. it's test case.
> But "Authentication error" happened. It looks like no user define in
> /etc/passwd.
> It did not ask the password.
> ----------------------------------------------------------
>   <users group="users">
>         <user name="linux" pwd="" home="/home/linux"/>
>         <user name="root" pwd="" home="/root"/>
>   </users>
> ----------------------------------------------------------

First of all this users section would put root into the
group users. I think this is not a good idea. I found this section
in the current liveCD description and changed it accordingly.
I recommend to do that too

	<users group="users">
		<user name="linux" pwd="" home="/home/linux"/>
	<//users>
	<users group="root">
		<user name="root" pwd="" home="/root"/>
	</users>

> I checked in ram image.
>  - /etc/passwd, /etc/shadow, /etc/group, /etc/nologin
>  - PAM configuration
> 
> It looks like no problem. What is the check point to investigate?
> There is no problem when created openSUSE10.3 LiveCD.

From the section above both users should be able to login
without password. For the root user this is not possible except
you call something like:

	passwd -d root
	pam-config -a --nullok

To check at least the linux user you can chroot into the
prepared tree and call su there

	chroot /path/to/image/root bash
	su linux
	su root

Last but not least:
The package resmgr must be installed

Hope that helps
