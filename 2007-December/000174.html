> Sorry, I meant qemu -hda /dev/sda of course ... this is the way I am
> checking the other bootstick systems.

The usbboot initrd was made for USB sticks only so its first task
is to search for a USB stick. The qemu -hda emulation emulates a disk
and no USB stack that's why it failes. There is another boot type
called oem (oemboot) which is used to deploy a preload system onto
a disk no matter if this disk is a USB stick or a "real" disk

> Out of curiosity - why is there a separate boot partition for the
> opensuse sticks?

The boot image and the system image are different entities. In your
case the system image is a read-only squashfs filesystem which you can't
access via the grub bootloader. According to this a single boot partition
with the size of the initrd+kernel+ext2-fs overhead is created and used
for the boot process
