>
> I could install the system as described in the Kiwi Homepage.
> When I start a Client it stops booting at:

What is the client specification? kiwi-ltsp is tested on very few
hardware, minimum RAM requirement is 128MB, but ideally it requires
256MB. Try booting few different  clients (even pxe booting laptop
will do), and see if the setup is working on at least one of them.
>
>  Starting X Font Server
>     /usr/lib/ltsp/screen_session
>
> the path /usr/lib/ltsp  does not exist in my system

It exists in the chroot image. However X font server is not your
issue, screen_session runs LDM (LTSP Display Manager). To find out
what is going on in the client at that time, ssh into the client as
root, password is linux.

>
> I tried to work  lts.conf  with:
>
> USE_XFS            = Y  und  N

You need to change /srv/kiwi-ltsp-nfs/etc/lts.conf and run the
following every time you change lts.conf:

rm /srv/kiwi-ltsp/ltsp-suse-10.3.i686-0.0.2
mksquashfs /srv/kiwi-ltsp-nfs /srv/kiwi-ltsp/ltsp-suse-10.3.i686-0.0.2
