Hmm, at least on all kernels I use the squashfs module is part of
a directory. For example in the 10.3 kernel it's

	/lib/modules/2.6.22.13-0.3-default/kernel/fs/squashfs
	/lib/modules/2.6.22.13-0.3-default/kernel/fs/squashfs/squashfs.ko

I guess you compiled your own module ?	

> and module is inserted to correct directory. Also add module name to
> INITRD_MODULES in sysconfig/kernel.
> But "Failed to mount RO root filesystem".

The INITRD_MODULES variable just stores the modules which should be
part of the initrd created by the distribution tool named mkinitrd.
The code in there will not be able to mount a root system based on
squashfs and the subsequent overlay aufs mount.
