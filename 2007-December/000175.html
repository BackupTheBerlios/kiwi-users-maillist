Hi Fruit Lovers :)

here comes the Christmas edition. This version includes
some major changes/enhancements and bugfixes reported in
bugzilla

Bugzilla:
=========
    - added cleanup code for temporary resolv.conf (#349491)
    - resize ramdisk filesystem to get full access (#348316)
    - fixed searchSwapSpace / searchDiskSpace function to return
      after first match (#349123) 
    - fixed umount of iso loop mounted paths (#348722)
    - fixed swap file setup if filesystem type is unknown (#348623)
    - added /usr/share/doc/packages/kiwi/ChangeLog (#349492)

Enhancements:
=============
    - added URL quoting according to RFC 2396. This allows to place
      username and password within an URL if credentials are required.
      Problem is that password as URL quote is human readable
  
    - added KIWIUtil.pm module used by Jan-Christoph Bornschlegel
      This is a collection of functions Jan needs for his job in kiwi
      but others may find it useful too. Please be aware the api in
      there is not stable yet

    - moved schema description from xsd to RelaxNG (#348084)
      Together with Thomas Schraitle <toms at novell.com> we created
      a more future proof scheme validation process. The kiwi scheme
      is now written in the compact RelaxNG format. This gives us
      the following advantages:

      a) this format can be converted very easily
      b) changes can be supported by an XSLT stylesheet and we can
         automatically convert old schemes into new ones
      c) documentation is placed into the scheme and will be extracted
         for our users

      Thanks to Thomas for his great ideas and help in that area

    - added first version of SaT module (#348734)
      With future versions of openSUSE we will integrate a very very
      fast dependency solver in our distribution. With kiwi I want to
      benefit from it and so I provided a first version of a perl
      binding for the satsolver. As soon as our distribution provides
      the solvable format natively I will start and integrate it. This
      will allow us:

      a) estimate size of an image very quickly
      b) estimate time to build
      c) solve patterns
      
      This is work in progress. If you want to have a look at the
      binding code you'll find it in tools/satplugin

    - added baseSetupBusyBox function (#349181)
      which allows you to replace the mentioned programs in busybox.links
      by a link to the busybox binary. This function should be used in
      boot images only and can reduce the size of your boot image.
      Using this function requires the package busybox to be installed as
      part of the boot image config.xml. Maybe this is interesting for
      our ltsp guys :-) Thanks to Petr Ostadal <postadal at novell.com>
      who implemented the function
      
I wish you all a merry Christmas and a happy new year
Hope to see you soon
