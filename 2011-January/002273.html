Hi,

> > But I don't think it's the kernel. I think it's just an incompatibility
> > between the kernel and the squashfs version. I think you build the
> > image with a mksquahfs which is not compatible with the kernel actually
> > mounting the filesystem
> 
> Yes, an incompatibility would make sense, question then is how to avoid it.

You can add a check like this:

   strings root/lib/modules/.../squashfs.ko | grep "squashfs: version"

and compare the kernel module version with;

   mksquashfs -version | grep "mksquashfs version"

both need to be at the same major version. In my case "4" if that
doesn't apply kiwi should stop. The solution is then to install
the appropriate package which is able to create the required
squashfs filesystem version

The buildsystem doesn't have to mount this squashfs before it is
put on the install image. That means you should be fine with just
the right creation tool.

I don't hope that squashfs will change it's file
format that often
