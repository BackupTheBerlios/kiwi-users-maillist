Hi,

> I've been using the same config-yast-autoyast.xml file for a while and the desired functionality doesn't seem to be working with kiwi 4.71.
> We are using it to configure ETH0 and ETH1, which has been working (I'm pretty sure it worked through release 4.70)
> I can see from my /var/log/boot.kiwi log that there was no /var/lib/autoinstall/autoconf/autoconf.xml file.  We use the kiwi-hooks to "sed" in some changes before allowing the system to "boot", and the hook script fails because the file does not exist.
> Prior to 4.71, we would see an automated AutoYast run, and it would set up the network.  Now the network is not set up, AutoYaST runs and we get an interactive session that makes you set the root password (which should already be set - from the kiwi config.xml ).
> Is there anything new with how we should be using the config-yast-autoyast.xml config file?  Or is this possibly a new bug?

Hmm, I received a patch from my suse studio team which did a change
in that area but it shouldn't have the effect you found :(

Basically the change was that we check if there is a 

  var/lib/autoinstall/autoconf/autoconf.xml

already existing in your image root directory. If such a file
exists we will not overwrite it with the version you have in your
image description. Would you mind to check if there is such a
autoconf.xml in your image root tree ?

The log file of the kiwi create step would also be helpful
could you attach it ?
