Hi,

> you can swich off boot graphics by just not installing the
> gfxboot branding packages. In your config.xml you will find
> something like:
>
>   <package name="gfxboot-branding-openSUSE" ...../>
>
> simply remove that line and there is no graphics boot anymore
It seems to work up to one thing. Even with no gfxboot there will be a 
line

ui gfxboot bootlogo isolinux.msg

still present in isolinux.cfg. The problem now, that even if a default 
label is set, it wont be loaded automatically (after a timeout), I always 
get the boot-prompt back after the time is over. If I delete this line, 
than it works: after timeout the default option gets loaded.

I think the 'ui' overrides the 'default' in the config file.
