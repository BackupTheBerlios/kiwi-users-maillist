> I'm trying to get the system to instead of downloading packages from the
> internet, which risk timing out and failing the process, to try and get it
> to load the files needed off the openSUSE 10.3 DVD.

> The following is how my repository is setup in config.xml file.
> 
>  
> 
>         <repository type="rpm-dir">
> 
>                 <source path="/media/SU1030.001/suse/i586/"/>
> 
>         </repository>
> 
>         <repository type="yast2">
> 
>                 <source path="opensuse://openSUSE:10.3/standard/"/>
> 
>         </repository>
> 
>  
> 
> Now for some reason the KIWI system still is downloading the files from the
> second repository listed above and not from the DVD.

The package manager always decide for the latest package. I guess the
opensuse:// repo provides newer or updated packages concerned to the
delivered media. What I don't know is what choice is made if the
same repo one remote the other local provides the exact same packages.
I think the packagemanager should choose the local one because of
speed reasons but I don't know exactly. If that depends on the order
of the repos it might help to specify it vice versa in the config.xml

> Are my repository statements in the proper order and syntax? (I tried to
> find an example document for different repository types and could not find
> one.)

you should at least change the first one to speed up things. if the
packagemanager has to calucalte the dependency tree from the RPM's
which it needs to do if you specify only a rpm-dir without any meta
data the process takes much longer. So I would change that to:

	<repository type="yast2">
		<source path="/media/SU1030.001/"/>
	</repository>

> Also what type of encryption is used to make the passwords necessary for
> setting up the users password in config.xml? So what type of encryption is
> this $2a$10$fq4yOF75pQY0lKjZkG2DBOdOlptomWELi0Gxd71pziLMGk4ZGIRH2

the user passwords are created by the crypt system call

	my $pwd = crypt ($word1, $salt);

see "man crypt" for details on the algorithm. It also depends on
what crypto algorithms are available on your system
