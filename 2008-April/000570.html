Hi David,

> I'm using a kiwi OEM image description.  I would rather let kiwi
> automatically determine the size of the resulting raw image file (by not
> using a <size> element in the <preferences> section), but I would like
> the partition to be larger in the resulting deployment.  Is there a way
> to specify the installed partition size somewhere, as is done with the
> netboot image with the PART=size;type;mount-point parameter?

you can influence the partitioning if you create a file named
"config.oempartition" and put that file into the oemboot boot
image description of your choice. As this is a change to a
user defined boot image I suggest to first create a copy of the
standard boot image. For example:

	cp -a /usr/share/kiwi/image/oemboot/suse-10.3 \
          /usr/share/kiwi/image/oemboot/suse-10.3-david

    cp config.oempartition /usr/share/kiwi/image/oemboot/suse-10.3-david/root

    ===> change system image config.xml:
         <type ... boot="oemboot/suse-10.3-david">oem</type>

the contents of the config.oempartition could be as follows:

	OEM_SWAPSIZE=<number in MB>
    OEM_SYSTEMSIZE=<number in MB>
    OEM_WITHOUTHOME=1
    OEM_REBOOT=1

I think what you want is only

    OEM_SYSTEMSIZE=<number in MB>

We also should add that into the documentation. I really need
to take care for the docs now :)
