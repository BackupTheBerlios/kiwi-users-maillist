> instead of plain <package> tags.
> This is the resulting error message:
> 
> Apr-25 23:41:22 <1> : Setting up SaT solver...
> Apr-25 23:41:22 <3> : --> No SaT plugin installed                    failed
> Apr-25 23:41:22 <2> : SaT solver setup failed, using generic module  skipped
> Apr-25 23:41:22 <1> : Including pattern: kde
> 
> Where can I find this "SaT solver" or documentation about this?

the satsolver is the new package solver which is available the first
time on SUSE 11.0 which doesn't exist yet :-) kiwi is already prepared
to use the satsolver for solving the patterns into a list of packages.
I have implemented support for it as a plugin because I don't want
you all to install suse 11.0 in order to be able to solve patterns :)
Therefore of no plugin exists the old (my own written) pattern solver
code is used

It's not problematic if you don't have the sat plugin. The advantage of
the satsolver is that it is incredibly fast. The plugin wil automatically
appear in the kiwi packages of the distribution which provides support
for it.

If you want to read the documentation of the satsolver you can
try to install the libsatsolver-devel package from one of the
zypper (yast package manager) backport packages:

	http://download.opensuse.org/repositories/zypp:/Backport/
