Hi Axel,

> coming back to the split image "COMBINED_IMAGE" option of KIWI. Do you 
> think using a data (read only) partition with a Squashfs would also work 
> for me?

sure if you setup your split section in a way that it contains your
data you can think of this read-only part as your data part

> IMAGE=/dev/sda2;my-image;1.0.0;192.168.150.1;8192, \
>             /dev/sda3;my-data;1.0.1;192.168.150.1;8192

kiwi would create my-image and my-image-read-only you can rename them
but I don't recommend to do that because it is in fact not only a data
partition but an esential part of the operating system

> PART=512;82;swap,10000;83;/
> DISK=/dev/sda
> COMBINED_IMAGE=yes

ok

> Any recommendations?

I would do it that way if the data can be read-only
