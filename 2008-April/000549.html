Marcus Schäfer schrieb:
> Hi Axel,
>
>   
>> beside the split image option in KIWI which is designed for read only 
>> partitions, does KIWI support something like a "data-only" image. For 
>> example loading a tgz archive file via tftp which contains a specific 
>> application which shall be updated on the installed system at a specific 
>> path, to avoid transfering and installing of the whole KIWI OS image?
>>     
>
> kiwi does not support data only images but I think there are several
> possibilities to get your archive onto the image after it has been
> deployed. For example you could think of an init script which you provide
> as a service (insserv) in your system image description. When the
> image boots the service runs and you can download the archive.
> Alternatively you can provide a package with the application somewhere
> and use yast2 firstboot to install it at boot time.
>
> Regards,
> Marcus
