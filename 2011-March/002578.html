> I don't have a good solution here. You should make sure the OS
> runs on a machine with lots of RAM and the stick hardware
> is fast. kiwi uses 7/10 of the ram space according to
> 
>   /proc/meminfo | grep MemFree

I should make myself more clear here. 7/10 of RAM are reserved
for the OS as writing space. The rest of the RAM is available
in the original meaning of RAM space... for loading applications
shared mem, etc...

This means if your system has 1000M of RAM 700M are used as
additional writing space for the root filesystem and only
300M are available for applications like your video streaming.
If the system starts swapping this will happen into the 700M of
reserved writing space (ramonly image). This means if the system
starts to swap it will run out of space sooner or later and the
kernel starts to kill processes if still possible. As soon
as the system starts to swap I would expect a perceptible
performance loss.

If you build the image without ramonly set the situation
should be better... but it's not what you want I guess
