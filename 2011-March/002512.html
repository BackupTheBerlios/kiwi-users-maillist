the image="usb" is gone and won't come back. You can have entirely
the same thing with the oem type and more. I don't see a reason to
maintain two code bases where one is just an extraction of the other.
Basically usb has survived only for the reason to create a custom
partition table which allows to have another partition than the
root partition as first partition. The only reason people requesting
this was to be able to use the stick as OS _and_ as data container
on Windows and Linux. Exactly that has been achieved with the recent
changes to the oem type. What most people dislike with the usb
type was that you need linux and kiwi to deploy the image on the
stick. One had to plug in the stick and call kiwi --bootstick in
order to deploy to the stick. So the type really wasn't very
popular and almost all people prefer to use imageWriter which also
brings you back to the oem or iso type
