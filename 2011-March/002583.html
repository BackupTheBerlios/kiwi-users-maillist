> I mean that
> live images are read only anyway right?

No, a live image on a CD/DVD is read only because the media is read 
only. A live system on a USB stick is no different than a system on a 
hard drive built into a computer. The stick is writable, thus your image 
is read/write. The only thing that makes the image on the stick read 
only is the "ramonly" setting in config.xml

> So would that change the fact
> that currently the os is sort of "resets" back to the original image
> when the system is rebooted?

Yes it would, see above, as changes are now persistent.

> If the "ramonly" element was changed
> would the OS then be able to save the changes made by a user?

Yes, without ramonly="true" the system on the stick is read/write and 
changes are persistent.
