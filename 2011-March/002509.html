Hi,

> Why with the OEM type image is the first boot with image any different
> than any other boot?  With the ISO type image every boot is the
> same...right? 

The oem image is an expandable image. on first boot it expands
itself according to the given oem-... parameters in your config.xml
If you for example told the oem to grow to a systemsize of 2G it
will try to do that on first boot. Of course that only can be
done if the size of the stick allows this. There are several 
oem parameters for details refer to the docs

> How is it with a Live image using and OEM image type
> that it "knows" that it's the first time it's being ran compared to
> any other time the image is booted?
> So that means if I configured that .desktop file on the users Desktop
> to point a /media   that should work but on after the first boot and
> not the first?

you can't compare a live iso with an oem image. Of course both have
the same root filesystem contents but their geometry is completely
different. a live iso packs everything into an iso filesystem
and a oem image is a virtual disk.
