On 03/22/2011 02:27 PM, Chris Cunha wrote:
> Guys I has attempting to add wireless support for Broadcom wifi cards.
>   I inserted"
>                   <package name="b43-firmware"/>
> 		<package name="broadcom-wl"/>
> 		<package name="broadcom-wl-kmp-default"/>
> 		<package name="broadcom-wl-kmp-desktop"/>
> 		<package name="broadcom-wl-kmp-pae"/>
> 		<package name="broadcom-wl-kmp-xen"/>
> for the drivers.  I also had to add the
> <package name="kernel-pae"/>
> as the Broadcom drives seem to require it.  Since the "kernel-pae" is
> in the update repo I had to add this repo to the config file as well.
>
> The image build was successful however on first boot of the image I
> noticed that the Broadcom drivers for this card was still not working.
>   I decided to reboot and now was able to see the list of new kernel
> boot options.  Since the system attempts to boot from the last kernel
> installed it was trying to boot from "xen" kernel and I received a
> "/boot/linux.4: file not found" upon boot.  The only options in the
> grub like list of kernels that does boot is the original (first one on
> the list) default one that I've been using right along.  What is the
> method to making whatever kernel is the latest it's one and only
> default?

You should have only one kernel in your appliance. I suggest you only 
use the default kernel, get rid off

<package name="broadcom-wl-kmp-desktop"/>
<package name="broadcom-wl-kmp-pae"/>
<package name="broadcom-wl-kmp-xen"/>

In your config.xml. These would obviously pull in the pae, desktop, and 
xen kernel packages.
