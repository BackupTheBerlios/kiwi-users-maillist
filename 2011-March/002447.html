> update:
>
> as consequence of the new --fat-storage option I'm going
> to remove the image="usb" type. It's entirely replaced
> by oem or iso types now. So in order to create the Windows
> friendly stick the following is supported
>  
> *) <type image="oem" filesystem="ext3" boot="oemboot/suse-11.3">
>        ... optional oem and systemdisk settings
>    </type>
>
>      kiwi --prepare ... --root /tmp/kiwi-11.3
>      kiwi --create /tmp/kiwi-11.3 -d /tmp/mytest/ \
>           --type oem --fat-storage 300
>
>    * This example creates a virtual oem disk (expandable) and thus
>      can be deployed with dd or with Matt's image writer tool on Windows too
>      on first boot the stick is resized according to the oem options
>      set in config.xml
>    * The --fat-storage option allows to specify the size of the fat
>      partition usable by Windows as data container (default 60M)
>      in this case 300M will be used
>    * fat partition is first partition
>
> I also update the documentation
>
