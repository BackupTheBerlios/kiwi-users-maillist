Hi,

> The way things currently work the fat partition contains a "boot"
> directory that holds the critical boot files for the OS.  As of right
> now I don't see a way to keep the end user out of the boot folder.

yes sorry that's not possible with a fat partition. fat fs doesn't
have a mechanism to hide folders

> Due to the boot folder being on the fat partition and there doesn't
> seem to be a way to create a "Storage" folder on this fat partition
> and have the .desktop file point to it, it seems that the .desktop
> file must be pointed to the /media folder.   If I'm clear on this,

I already wrote you how you can create a persistent mount
point for the fat partition. You can enhance the script to also
create a Storage folder and let your .desktop file point to
that folder

   ....
   mount ${bootdev}1 /media/stickstorage

   mkdir /media/stickstorage/Storage
   
let the .desktop file point to /media/stickstorage/Storage.
But this will still allow the user to access /media/stickstorage/boot
sorry this can't be prevented
