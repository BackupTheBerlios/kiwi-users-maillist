> I am trying to add an 'install on HD' option to the boot menu of a Live image built with kiwi (like in the official installation DVD).
> 
> I was expecting that adding this line:
> 
>     <package name='yast2-live-installer'/>

This just adds the yast2-live-installer package. You can can install the
image from the running LiveSystem.

Kiwi does not support the creation of additional entries in the grub
menu through the config file. I have thought about whether or nt this
would be useful and the traps that would open if we did support such a
feature.

Anyway, even if you could add an 'install on HD' option, it would still
not be quite as straight forward. The install mechanism for the 'install
on HD' option requires the presence of the packages, i.e. on you DVD you
will need to have the *.rpm files.

Kiwi does have a product creation mode, that is used to create the SUSE
install media.
> 
> to the control.xml in the section :
> 
>     <packages patternType='onlyRequired' type='image' patternPackageType='onlyRequired'>

The patternPackageType attribute is obsolete, you can remove it from
your config.xml file.
