Hi,

> >To put another way, if I prebuild my boot image, will setting the "bootinclulde" attribute in my system image config.xml have any effect?
> 
> Marcus will need to answer that one.

if you prebuild it and after that you set some bootinclude statement to
a package or archive it will not have any effect on the prebuild boot
image. That is because kiwi just use the prebuilt version. In such a
case you should deactivate the checkprebuilt attr and activate it
again after kiwi has built the new version of the boot image

> >Does branding need to be deployed as an RPM in order to work?
> 
> No, it does not, see example above.

But we recommend to use a package. This is the more clean solution
the branding system als has changed from one distro to the other.
This means if you create an archive you have to make sure it installs
the right data to the right place, which requires some knowledge
about branding in openSUSE. If you clone from an existing branding
package and change the theme and package name as well as add your
own branding files according to the package contents you could be
sure it will work for the distro version this branding package
was provided
