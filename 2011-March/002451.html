Hi,

> I have a very important requirement that I have to ask about.  The
> way that I'm currently using my kiwi build image is using a live ISO
> image.  Meaning that if any changes are made to the os while using
> it on the USB drive are lost upon the next reboot.  For our purpose
> is very important that we have this feature. Is the OEM image type
> that you described above going to be able to work that way?

I added that feature:
If you set the following type setup it will be the same as for
the iso but with a usable fat partition for Windows

  <type image="oem" filesystem="clicfs" ramonly="true" boot="oemboot/suse-11.3">
    ... optional oem and systemdisk settings
  </type>
