On 03/29/2011 06:35 PM, Mark Haase wrote:
> I have been reading a lot of docs and source code... slowly Kiwi is making sense to me, but there are a few mental hurdles I just can't get over on my own.
>
> (I am running Kiwi 2.82 and OpenSUSE 11.4.)
>
> 1. Bootinclude attribute: Does this mean that kiwi will modify the raw boot image and installs new packages into it?

No.

> Or does it mean that when kiwi builds the boot image (as a subroutine while building the system image) that it will add those packages at that time?

Yes, building the initrd (boot image) currently happens via recursive 
call of kiwi onto itself with the parameter set such the control file of 
the specified boot image is found. With this a prepare and create step 
is run to create the initrd. packages and archives marked as 
"bootinclude" get bundled up in the initrd.

>
> To put another way, if I prebuild my boot image, will setting the "bootinclulde" attribute in my system image config.xml have any effect?

Marcus will need to answer that one.

>
> 2. Bootdelete attribute: The function of this is [mostly] clear to me, but I don't understand the use case. For example, branding packages are marked as bootdelete="true", but why?

Because we do not want the package itself, we want the installed 
content. bootdelete=true gets rid of the package, i.e. the rpm file, 
also applies to archives.

> Aren't they needed in the final boot image?

Yes, tey are, but we are interested in the installed content, not the 
package itself.

>
> 3. I am trying to set up branding by putting my artwork and configurations in the overlay (bootsplash in etc/bootsplash... and gfxboot in usr/share/gfxboot...). Is that a mistake?

No, this is possible. You can find an example here:

http://en.opensuse.org/SDB:KIWI_Cookbook_Splash_Screen

The example uses an archive, but you can create the setup directly in 
the overlay tree.

> Does branding need to be deployed as an RPM in order to work?

No, it does not, see example above.

> (I am including gfxboot and bootsplash in my system image, but not including any gfxboot-branding-* or bootsplash-branding-*).

Don't forget to set <boot-theme> in config.xml and make sure you use a 
directory other than the default.
