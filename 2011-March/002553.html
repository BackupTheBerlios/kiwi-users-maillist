On 03/25/2011 10:49 AM, Marcus Schäfer wrote:
> Hi,
>
>> Guys this fat partition that we created on the USB drive...is this
>> FAT16 or FAT32?
>
> FAT16
>
>> I ask because Fat 16 looks like it has a file storage limited of
>> either 2gb or 4 gb depending on the OS it's running on and Fat32 it
>> seems to have 32GB for all OSs and 2TB for some OSs.
>
> correct
>
>> If we are using FAT16 is it nothing more than changing something in
>> Kiwi to make it use Fat32?
>
> in /usr/share/kiwi/modules/KIWIBoot.pm, search for
>
>     $partid = 6
>
> and change it to
>
>     $partid = "b"  (That's W95 FAT32)
>
>> I just figured that the more storage on the USB drive the better but
>> I'm not sure of Fat options with Kiw/Linux.
>
> Frankly said I wouldn't change this because I don't know how
> syslinux reacts if a fat32 partition is used. Maybe the system
> doesn't boot anymore but that needs to be tested. Other than
> that we are talking about a usb stick and imho nobody will store
> files greater than 4G on this storage device

Well, 64k turned out not to be enough memory, thus I'd say we'll see 
people try to stuff very large files onto USB sticks sooner rather than 
later. That said, we need to focus on the technical implementation. The 
first concern would be, as Marcus pointed out, the behavior of syslinux 
on a FAT32 file system. The second part would be the behavior of 
Windoze. After all, the FAT storage partition only exists as a means to 
share files with Windoze and I for one have my doubts that Windoze will 
handle files >4GB that are stored on a FAT partition correctly.

my $0.02
