Hi Rahul,

> what is the update mechanism for Thin client images of type "split"

if you use a PXE deployment method each image portion exists
separately and can be updated separately (netboot takes care for that)
but in case of a virtual disk (oemboot) the image is represented
as _one_ disk which makes a partial update difficult.

> Is there a way to update only the RO partition while leaving the
> RW partition untouched. ? If not, which other alternatives are available.

if you use a network deployment yes if you use a virtual disk
solution the data of your deployed image somehow needs to become
part of your virtual disk image. At the moment there is no workflow
for that but it's a good point to start a discussion. With a virtual
disk it's the same situation as if you install a new harddisk on
a computer. Somehow the data from the "old" harddisk must become part
of the new disk
