Hi,

> while testing  I made some wrong entries in my xml-files. No problem
> because of backup-files ;-)
> Is there a way to check the config-files before starting the building to
> save time?
> Or better can kiwi check all files before starting.
> In my case a < was missing an kiwi stops the build after 10 minutes.

that's because you made changes in boot image descriptions which are
build later. The validity of a system image config.xml is checked
directly at the beginning. You can manually validate a config.xml
with the following call:

	jing /usr/share/kiwi/modules/KIWIScheme.rng config.xml
