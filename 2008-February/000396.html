Is there a way to configure the size of partitions while creating an oemboot image.

I am trying to create a "split" type "oemboot" image.  By default this image requires a Swap partition of at least 50 MB. Because of this my image runs out of disk space during the 2nd stage of deployment.

I have a 512 MB Flash Disk and the  disk ussage of various partitions is as follows

kiwiboot - 31 MB
RO partition - 376 MB
RW partition - 101 MB.

I tried using  OEM_SWAPSIZE in "config.oempartition" to control the SWAP size but this seems to be ineffective.

I need to find a way to be able to configure the size of either the RO, RW or SWAP partitions to get the image to fit into 512 MB.
