Hello. I've been using kiwi to build LiveUSB sticks
for a while. My workflow broke today with an update of
kiwi and I haven't been able to find any way to get it
working by changing my usage. I had to hack kiwi to
get it working.

This is the error message:
Feb-19 15:25:14 <1> : Creating boot USB stick from:
/myth/notmyth/kiwi/kiwi-suse-10.3-custom-KDE/initrd-usbboot-suse-10.3.i686-2.1.1...
Feb-19 15:25:14 <3> : Failed to loop mount system
image: mount: you must specify the filesystem type

Adding "-t squashfs" to the mount command at line 376
of /usr/share/kiwi/modules/KIWIBoot.pm took care of it
for me. I don't know whether that's a proper solution,
i.e. whether squashfs is the only possible fs type
there.

I don't know what is the latest version I had working
before today's update, but it wasn't very old. The
current installation reports version v2.27 SVN:
Revision: 1061. It is the latest binaries available
from the build service repo for opensuse 10.2: 
kiwi-2.27-28.1
kiwi-desc-usbboot-2.27-28.1
kiwi-desc-isoboot-2.27-28.1
kiwi-desc-livesystem-1.59-25.1
kiwi-tools-2.27-28.1
