thanks once again for your fast reply. Your patch worked fine for me - now kbd gets automatically started and the keyboard layout is as configured.

Regarding the image size issue i think i understand what you mean, but from a users point of view it doesn't really matter. If I have a read-only media (e.g. iso) i want the image to be as small as possible and wouldn't specify a size manually. If I want a writable image (e.g. a vmx one) i would specify the size of my filesystem (cause i calculated beforehand how much space the data takes and how much more i would need to work) and expect the image to adapt so my filesystem fits in. As long as there aren't options for more partitions or so i fail to see the benefits - but perhaps my point of view is to naive and I'm missing something. Anyways, as long as it is configurable, I'm fine - although i would like to suggest to mention this in the docs.

Further I had to be root to run "kiwi --createpassword". That's not a big deal but I'm just used to do as much as possible as normal user and I fail to see a reason why this shouldn't work as normal user.
