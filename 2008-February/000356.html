Hi Marcus,

I tested your size patch with 2.28 and it worked just fine for me. Further i can confirm that your change regarding the size definition of an image works in 2.29 and that the kbd service gets started automatically too, so that the keytable tag is effective.

Thanks a lot for those changes!

Now the bad news aka what I found faulty in 2.29:

1. Calling "kiwi --createpassword" as normal user results in different output compared to calling it as root. Further I'm not able to login if I use the output created as normal user. If I use the one, I created as root, it still works fine. E.g. "kiwi --createpassword" for "linux" as normal user is "xx/OHzLpCzQE6" and doing the same as root results in "$2a$05$xlVe2u/mkxdx4vsECWcoi.qCqhodD5mMW..EVG8AAIUioIlq1lIFe".

2. Every single password I created as root started with "$2a$05$xlVe2u/mkxdx4vsECWcoi." IMHO this makes the password encryption more insecure because you thereby reduce the available room to ~50% (31^x instead of 60^x).

3. If i press <CTRL>+<C> while executing the "kiwi --createpassword" program it says "Received signal INT" and exits the program but from then on I don't see what I enter into the shell and pressing enter just prints the prompt again in the same line except if i entered some invisible text. This just looks a bit unclean.

4. Further I noticed that the smart channels used for installing the system (e.g. kiwi2-9602) are still left in the smart configuration after the installation. IMHO it would be better / cleaner to remove them because the are unusable anyways ;).
