Hi Tina,

> I'm trying to produce a working .vmdk image with kiwi but after updating to the latest available version for 10.3 from openSUSE:Tools (2.27-28.1) it wont start anymore and reports errors you can see here: http://pastebin.ca/909802 . My config.xml can be seen here: http://pastebin.ca/909814 .

It seems you have a checksum created (or accidently copied) in your
system image description. You should find a file called:

	config/suse-server-10.3/.checksum.md5

if this file exists kiwi checks if the md5 information matches the
description. but in your case it failed because the md5 information
contains md5 hashes for stuff like linuxrc which is only present in
boot image descriptions but not in system image description.

* I guess this md5 file exists there by accident. So the solution is
  to simply remove the mentioned file and try again :-)

* A first look into your config.xml tells me that this image
  description is a clone of a boot image description which you 
  now use as a system image description. That's not a good start
  Please do _not_ use any boot image descriptions from
  /usr/share/kiwi/image/*boot as base description for your
  own image. If you need something to start with refer to one
  of the examples in:

    /usr/share/doc/packages/kiwi/examples

  or use one of the templates used by others from here:

    https://forgesvn1.novell.com/svn/opensuse/trunk/distribution/images

Hope that helps
