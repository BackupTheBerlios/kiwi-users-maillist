Ah ok, there is something like that. For example:

	<keytable>de-latin1-nodeadkeys.map.gz</keytable>

this will setup the sysconfig/keyboard file correctly. I don't
know if you have read the mail from Tina. She also has problems with
the keyboard setup which are fixed in the latest kiwi version

>> I guess
>> the problem is the creation of the xorg.conf file. We use sax2 to do
>> the job and sax2 takes a look into the sysconfig/keyboard file to
>> map the console keyboard layout into the X keyboard layout. If it
>> runs X in English as you said I guess your console keyboard layout
>> is also in English. You can try to edit the sysconfig/keyboard
>> file and after that run sax2 again.
>>
>>   
> Hmmm, after I modified keyboard to the correct value now, the console has 
> German keys now, but in KDE, the US flag is still showing. Even if I switch 
> that to German, this setting is not persistent and at next startup, US 
> English is still the X-Default. Sax2 does not change this, so perhaps my 
> sysconfig is broken somewhere.

after you have changed the console keyboard setup you need to
recreate the xorg.conf by calling:

	sax2 -r

the -r switch tells sax to re-detect everything including the
keyboard environment. sax maps the console settings into X11 settings
then
