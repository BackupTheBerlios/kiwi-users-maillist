> Hi Robert.
> I thought that I removed any non-official repo from my config files
> list of repos.
> http://paste2.org/p/1370745
> I looked in my config.xml file and did a search for clicfs. I find
> nothing that specifically states what version of this package is to be
> used so why didn't the new version get used for the same repo as
> before?

zypper (the package manager) always pulls the latest available package 
from the available repositories. Thus, on Friday you got whatever the 
version was on Friday and today you got today's release. You need clicfs 
based on the image type you are building. clicfs in 
http://download.opensuse.org/repositories/filesystems/openSUSE_11.3 is 
newer than the version in opensuse://11.3/repo/oss/, therefore you get 
the version from the filesystem repo.

This is package management 101. ;)

> The package "liblzma0" isn't listed as well in the config.xml
> file so I assume it's being installed from one of the patterns that is
> listed.

No, liblzma is a dependency introduced by the package format itself. 
lzma is a compression algorithm (http://en.wikipedia.org/wiki/Lzma) that 
allows RPM to compress the package data and payload.
