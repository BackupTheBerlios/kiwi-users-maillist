> When you use a "ramonly" OEM image the image itself will load a lot of it's
> data into RAM and thus eat up chunks of RAM that cannot be used by
> applications running from the stick. Therefore applications are likely to
> start swapping quickly. Swapping has a negative effect on performance,
> especially on a USB stick as the transfer rate over USB is not the fastest.
>
> When you use an OEM image that allows read and write less of the system data
> is loaded into RAM, therefore applications will not start swapping as early
