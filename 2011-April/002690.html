This is the first since the change to the USB paradigm, i.e. since 
image="usb" went away that I tested the new deployment method and I have 
run into an issue. Using the following type definition i config.xml:

<type image="oem" boot="oemboot/suse-11.3" filesystem="ext3" 
bootloader="syslinux">
     <oemconfig>
	<oem-swap>true</oem-swap>
	<oem-systemsize>1024</oem-systemsize>
	<oem-silent-boot>true</oem-silent-boot>
     </oemconfig>
</type>

-> kiwi --prepare .... --root ....
-> kiwi --create .... -d ....

Then dd the raw image to the stick. Upon boot I would expect the file 
system to expand to take over the rest of the stick, but it doesn't. On 
the system running from the stick on a laptop I get the following 
information:

-> df -h
Filesystem            Size  Used Avail Use% Mounted on
/dev/sdb1             4.4G  3.0G  1.2G  73% /
devtmpfs              976M  260K  976M   1% /dev
tmpfs                 976M  7.4M  968M   1% /dev/shm
/dev/sdb2              98M   25M   73M  26% /boot

-> fdisk -l
Disk /dev/sdb: 8032 MB, 8032092160 bytes
4 heads, 32 sectors/track, 122560 cylinders
Units = cylinders of 128 * 512 = 65536 bytes
Sector size (logical/physical): 512 bytes / 512 bytes
I/O size (minimum/optimal): 512 bytes / 512 bytes
Disk identifier: 0x7013bf8d

    Device Boot      Start         End      Blocks   Id  System
/dev/sdb1               1       72232     4622832   83  Linux
/dev/sdb2   *       72233       73792       99840    c  W95 FAT32 (LBA)


Something is amiss here. I think we have a bug :(
