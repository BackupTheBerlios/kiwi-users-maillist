Hi,

> I need to make sure that an older version of
> a specific package is not overwritte by updates
> available in the rpm package sources.
> 
> This question has been asked before, but unfortunately
> it was not answered. See
> https://lists.berlios.de/pipermail/kiwi-users/2009-September/001443.html
> 
> does this work?
> <package name="name-version" ... />
> e.g
> 
> <packages type="image">
>   ...
>   <package name="fvwm2-2.5.21-69"/>
> </packages>
 

It works that way for the zypper packagemanager. Other package
managers like smart or yum might expect another name string.
The packagemanager documentation should clarify that. For zypper
you can just place the name of the package which is printed out
if you call e.g:

  rpm -q vim

  vim-7.2-23.1.i586

you can use this name or a shorten version like:

  vim-7.2-23.1
  vim-7.2-23
  vim

> Later:
> Works for me. Should this maybe mentioned explicitly in the Cookbook?

I think it's more targeted to what the packagemanager requires.
But it might be good to mention that the name string is passed
to the packagemanager and that the user should refer the
packagemanager documentation concerning full qualified package
names
