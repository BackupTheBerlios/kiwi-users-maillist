Yes, but a potential but valid use case is that there are good reasons (e.g. security issue, special feature, additional tracing support, ...) to have on the host a local version of zypper installed in e.g /usr/local/bin/zypper which might even internally call the real SuSE zypper from /usr/bin/zypper.

This would be totally fine and valid from a FHS point of view while breaking kiwi unnecessarily as I would expect kiwi to use the zypper binaries as provided from the zypper rpm.
