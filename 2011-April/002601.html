Hmm, this sounds more like a clicfs problem. Can you do the
following:

  * boot the machine
  * open a terminal window and type

    su root
    tail -f /var/log/messages

  * leave the terminal alone but visible and start to do
    what you did before (browsing, streaming video, etc...)
   
  * watch the messages window for errors, do you see any
    filesystem related information ?

Thanks

> When you mentioned "ram space" are you referring to the "oem swap
> space" in my config.xml file?

no, when I talked about RAM I meant the physical ram space
in your machine. From what you've said it looks like the
system doesn't swap at all. You said the CPU is at 100% and the
system is locked up. That's not normal for a system on swap.

> The primary purpose of this live image is to run an Internet browser
> on a variety of sites. That would be it's primary function.

I got that. Would it be a problem for you to make another
test without the clicfs filesystem:

   <type image="oem" boottimeout="2" filesystem="ext3" boot="oemboot/suse-11.3">  
Be aware after create the image will be much larger. But it's
for testing if we run into a clicfs problem here
