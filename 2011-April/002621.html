On 04/05/2011 12:33 PM, EXTERNAL Konold Martin (Firma, RtP2/TEF72) wrote:
> Hi Marcus,
> Hi Robert,
>
>>> Why not make the path to zypper and other tools implicit
>> instead of explicit?
>>
>> Hmm, I think you are right even though according to FHS
>> /usr/bin/zypper is the correct place and it should not change.
> Yes, but a potential but valid use case is that there are good reasons (e.g. security issue, special feature, additional tracing support, ...) to have on the host a local version of zypper installed in e.g /usr/local/bin/zypper which might even internally call the real SuSE zypper from /usr/bin/zypper.
>
> This would be totally fine and valid from a FHS point of view while breaking kiwi unnecessarily as I would expect kiwi to use the zypper binaries as provided from the zypper rpm.

This is the problem KIWILocator is supposed to address. As the user you 
can manipulate the path to find "your version of zypper" for example. 
Problem is how do we communicate this in the chroot environment. In the 
chroot we most likely do not have the users version of executable X and 
thus we must find the "default" location. Our encoding scheme of these 
locations has to change, this should be a general solution.
