Hi,

> Why don't keep majority of stuff on read-only layer,
> and create just a few symlinks to read-write layer
> for writable stuff?

We had that system before and it turned out to be not
flexible enough. It's really hard to identify a global
useful set of writable data which doesn't bite some
else requirements. The split approach at the moment
creates a symlink tarball which contains the read-only
and the read-write links. According to this you always
have a full writable system except for the files already
existing in the image. Of course all data you write are
temporary as long as you don't specify them as
persistent

> Currently kiwi uses opposite approach:
> read-write contains sea of symlinks
> replacing almost all files in system tree.

That's part of the code, _all_ files except the temporary
one living in RAM are symlinked to either a read-only or
a read-write destination

> Question, again, is about motivation.
> Why not change mind to opposite?

The motivation is to have a full flexible system. I agree
if you say it's hard to create a <split> section which 
includes all the information about temporary and persistent
data. 

At the moment I provide examples with <split> sections
templates to get people started. This section looks like
the following:

    <split>
        <persistent>
            <file name="/etc"/>
            <file name="/etc/*"/>
            <file name="/var"/>
            <file name="/var/*"/>
            <file name="/boot"/>
            <file name="/boot/*"/>
        </persistent>
    </split>

Some time ago I thought we could also implement a hard <split>
section containing the "most popular" read-write locations as
you suggested and use this default split information if no
split section was defined in config.xml

I think that we have both a full flexible system but also at
least a standard system working of no split information is there.
What do you think ?

If you think this is a good idea I need your and all other help
to dicsuss

   _what would be a good default split section_

Thanks
