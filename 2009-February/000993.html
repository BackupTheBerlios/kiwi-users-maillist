Roger,

> I did not have the bootprofile="smp" tag. So I added it. I also did not
> have the kernel-pae-base and aufs-kmp-pae rpms. So I added them. But
> things are still not working. I have attached the logs from when I made
> the images. Note that my logs also contain a line for each deleted
> kernel module, and a list of the modules that remain when my image.sh is
> run.
> 
> What happens now is that the boot prints "Booting KIWI..." and then a
> few lines later says "Network module: Failed to load network module !"
> 
> When I used kernel-default everywhere (and no bootprofile="smp" or
> aufs-kmp-pae) it works fine. I think the drivers are there.

To debug this you should pass the kernel parameter

   kiwidebug=1

in your pxe configuration. If the kiwi boot now fails you will
get a shell instead of a reboot exception and can debug what 
network kernel modules was about to become loaded and why this
failed
