Hi,

> Well Ok,
>
> I am using x86_64 arch so I had to specify it into the config.xml file.
> <packages type="bootstrap">
> 	<package name="cracklib-dict-full" arch="x86_64"/>

This only means use cracklib-dict-full on x86_64 only but due
to hard dependencies the package will be pulled for any other arch
too. So this has no real effect

if you have a 64bit build arch you also need a repo which provides
64bit packages. If you have a 32bit repo only you have to run kiwi
with 

   linux32 kiwi ...

and this will result in a 32bit image on your 64bit build machine
The other way round is of course not possible.

Setting the arch flag should only be done if you want to force
a package of a specific architecture and even in this case the
package of the desired architecture will only be used if the
repo provides it

> RTFM....
> ? Can I manage the different arch type with profiles ?

you shouldn't the packagemanager automatically picks up the
right packages according to the build machines architecture. This
requires the repository to provide these packages. So you should
not have any arch exception in your case

Make sure your SLED repo has a suse/<arch>/ directory and arch
is the same as provided by "uname -m"
