>> In my case the root is almost empty: It contains some

> If you start a system with just /usr and /boot it's broken

No, /usr and /boot are empty, but they have to exist.

Somehow simplified (only /usr shown here),
my kiwi usage looks like this:
> mkfs.ext4 /dev/sda1
> mkfs.ext4 /dev/sdb1
> mkdir /tmp/my_root
> mount /dev/sda1 /tmp/my_root
> mkdir /tmp/my_root/usr
> mount /dev/sdb1 /tmp/my_root/usr
> kiwi --prepare --root /tmp/my_root --dont-recreate-new-root
As you can see, /usr is pre-mounted before kiwi is called.
The same holds for /boot, /tmp, and
probably one or two other directories.
These directories do not contain any files, so
I don't think this leads to broken integrity.
They just need to be created because they reside
on separate filesystems on separate devices.

I have the feeling that we talked at cross-purposes here.
Do you still expect my setup to yield broken systems?
Otherwise, would you reconsider my proposal?
