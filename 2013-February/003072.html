Hello,

I would like to create an PXE image using kiwi where the pxe client
machine has no local disks. I started with the suse-pxe-client example
which works fine by using the default config.xml provided. Unfortuantely
this setup needs a local disk.

For the diskless setup I removed the partioning part of config.xml and
changed the the filesystem to ext3. The cotrresponding part of
config.xml now looks like this:

        <preferences>
                <type image="pxe" filesystem="ext3"
boot="netboot/suse-12.2" ramonly="true">
                        <pxedeploy server="192.168.10.1" blocksize="4096">
                        </pxedeploy>
                </type>
		...
        </preferences>

On the PXE server I modified tftpboot/KIWI/config.default file so that
it contains only this line:

IMAGE='/dev/ram1;suse-12.2-pxe-client.x86_64;1.2.8;192.168.10.1;4096'

Building the image is no problem. When the client boots up it loads the
config.default file and a short while after booting fails. The last
message I see is:

/sbin/init no such file or not executable

After 2 minutes the system then resets itself.

Doe anyone have a idea what goes wrong or what I have to change in the
configuration?
