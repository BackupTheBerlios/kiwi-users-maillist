Hi,

> During a recent discussion with an ISV interested in using Kiwi for 
> appliance building the question was raised about support for 
> non-encrypted passwords in the config.xml file. Technically this is 
> probably not difficult to support and since SUSE Studio uses 
> non-encrypted passwords in the interface it might be a nice feature to have.
> 
> For most appliances the appliance creator probably ends up sharing the 
> password with someone anyway, and for appliances where the password must 
> be protected the encrypted option would be available or just be the default.

I think that would be an easy patch in kiwi to use the pasword string
directly as "the password". So far we only support to add encrypted
passwords because I thought people don't want to see that in plaintext.
To support that we have to add an attribute which marks the password
as not encrypted because otherwise we can not determine if the text
is a pasword or a crypted hash.

Feel free to open an enh. report for this or even better send
a patch :-))
