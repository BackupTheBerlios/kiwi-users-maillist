> Sorry I don't get that. When I log in I type a plain text password. I 
> presume this is then encrypted somewhere and compared to what is stored 
> in /etc/shadow. So if the encryption is different how can the comparison 
> work?

To my knowledge there is a chain of different encryption algorithms
running. If one matches you can login. At least the standard crypt()
created passwords still works in /etc/shadow

Anyway I have changed the code to use blowfish now. You can see that
if you pull from the current master branch. This action now also works
with any user as it does not directly access the dm crypt pool like crypt().
The password is written as hex representation. For details see

  man Crypt::CBC  => encrypt_hex()

This also implies that I used the cipher block chaining mode (cbc)
together with the blowfish encryption from the perl-Crypt-Blowfish
package.
