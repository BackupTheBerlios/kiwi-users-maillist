Hi,

 

To my understanding the hashes contain a "marker" that determine with which
algorithm they have been created (DES no marker,  $2a$ for blowfish and $1$
for MD5).

So when the system needs to compare a plain password with the saved hash it
knows by looking at the hash which algorithm to use. 

The crypt() function on my system (SuSE-11.1) creates Blowfish hashes. So no
problem - I was just wondering if it is always under all
conditions/distributions/versions true.

My second concern was/is the salt. I think the salt should be true random and
not the userid. If the userid is used it is probably a lot easier to
"gues" the original plain password.

 

As far as I can see you don't need to be root to create theese hashes.
