On Wed, 2009-12-09 at 14:26 -0500, Robert Schweikert wrote:
> I think I have found a password creation/encryption problem. When I use
> 
> -> kiwi --createpassword
> 
> and use linux as the plain text password I get the following encrypted 
> string back:
> 
> $2a$05$vgmS7MrbVWrghZGWn84CpeDjVSEzc5NSrYb8wtkwkm4EjJ2mRN4cO
> 
> When comparing this to the encrypted password in the example config.xml 
> files ($1$wYJUgpM5$RXMMeASDc035eX.NbYWFl0) the difference is obvious. 
> But the example config.xml files also use linux as the password.
> 
> I would have expected to get the same encrypted value back from 
> createpassword as is encoded in config.xml.
> 
> The createpassword invocation ends up calling Perl's crypt() interface 
> which according to http://www.osix.net/modules/article/?id=571 
> implements DES. However, the default encryption algorithm for the distro 
> is Blowfish, I believe. I also think this changed in the not too distant 
> past.
> 
> Couldn't figure out how to use Crypt::Blowfish or I would have tested this.

The same password will be different. That is what makes it more tricky
to break. The same encoded string, however, is always the same password.
