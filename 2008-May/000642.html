> I've been able to do a successful build with 2.38-61, for split and 
> aufs mounts :-)
I found some difference in the new stick partition layout and I'm not 
sure if this was made on purpose.
The data r/w partition (now part2) is only "ext2" not "ext3" by default, 
while the old layout had an ext3 by default on part3.
Since there is no automatic fsck -y during reboot and before mounting 
aufs, this leads to problems if the r/w partition has errors.
I could either activate fsck on the stick (not recommended?) or manually 
add a journal during creation of the bootstick, but ext3 by default 
might be a cleaner approach?
