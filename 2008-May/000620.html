Hi All,

I have updated the documentation and would like to ask
you for your feedback. Hopefully you'll find the document
useful.

Luis according to your last mail we mainly discussed the
issue of the target audience. We agreed on the following:

1) Quickstart
   For users who basically want to build one of the
   provided sample images with little or no modification.

2) Cookbook (former design draft)
   For developer and you call it "power users". people who
   want to make the most out of kiwi capabilities.
   * describes how kiwi works and references all possible
     image types as examples
   * adds flavours of each image type which are the changes
     power users want to make

I'm sure there could be many more valuable information added.
According to your feedback I would like to identify the most
important things which are still missing.

I think we also should have in mind that in principal every problem
around the operating system is combined to one place if you create
an image. So it seems not to make sense to describe how to setup
X in an image or how to setup sound support in an image,
it would be a never ending list :) Please correct me if I'm wrong
but such information seems to be useful to be present in a wiki
and can be added by everybody who solved such problems.

Let me know what you think

Thanks

Regards,
Marcus
