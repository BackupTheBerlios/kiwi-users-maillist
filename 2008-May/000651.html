Hi,

> I am currently building a LiveCD based on oS 11.0 and ran into a problem during boot of the created ISO image:
> 
> Have size proc/meminfo -> 5 MB
> Need size: /read-only/rootfs.tar -> 500 MB [ uncompressed ]
> 
> I am trying to boot the ISO in VirtualBox.

not enough RAM assigned for a ramdisk in your guest system

> What happens in this stage? Is the complete rootfs mounted into ram? Normally the squashfs image should just be mounted and then re-mounted rw using unionfs (aufs?).

I guess you have set flags="compresed" in your config.xml. In this mode
kiwi creates a splitted image and uses the information of the split section
to create a tarball for the data you have specified there.

If you want aufs to be used just change the flags attribute to

	flags="unified" 

and rebuild.
