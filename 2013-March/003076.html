On 03/11/2013 10:50 AM, Evgeny Podjachev wrote:
> Hi, I build 64 bit distribution based on opensuse 12.2 using kiwi
> v5.04.59. But when I try to add 32bit rpm to it I get the following
> message:
>
> Overall download size: 195.0 MiB. After the operation, additional 731.8 MiB
> will be used.
> Continue? [y/n/?] (y): y
> Installing: aksusbd-2.1-1 ...............................................[error]
> Installation of aksusbd-2.1-1 failed:
> (with --nodeps --force) Error: Subprocess failed. Error: RPM failed:
> The 32bit support is missing. Please install the x86 compatibility
> packages required by your distribution and retry the installation.
> See the installation guide for more details.
> error: %pre(aksusbd-2.1-1.i386) scriptlet failed, exit status 8
> error: aksusbd-2.1-1.i386: install failed
>
> But I easily install this package on deployed system.
>
> Is it possible to mix i386 and x86_64 packages in one distribution?

Yes it is but as the error message indicates you need to install the 32 
bit runtime environment.

The package, rather than having a Requires: appears to check the 
existence of the 32 bit RTE (Run Time Environment) in the %pre snippet :(

I think there is a pattern for the 32 bit RTE, but I do not know the 
name. You'll have to dig for it ;) alternatively installing the 
"linux32" package may pull in enough to get you going.
