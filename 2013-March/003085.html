Hi,

> > As said in my initial posting, I actually didn't use kiwi for
> > system deployment until ~one month ago, I just called zypper
> > with --root.  kiwi does the same thing, but better, e.g., I
> > forgot about initializing /etc/{group,passwd} in the new system
> > before calling zypper, which broke some file ownerships in my
> > systems.  I could simply mimic the way kiwi performs its image
> > perparation, but I feel this is unnecessary work and I might
> > just call kiwi for this.
> Although I met some resistance, I'd like to offer
> this patch, which adds an option "--force-bootstrap"
> and enables bootstrapping in "--recycle-root" mode.
> This is what I'm currently using on my machines as
> I can't avoid preexisting subdirectories in my root.
> If you can bring yourself to accept it, I
> will add some lines of documentation.

Sorry for the late response. The patch looks good to me. I added
at least a few lines of usage message and will push the result
after the testsuite completed. here is the commit:

commit 3384a30ffd6366da231fca2a68f5aca8eb376d07
Author: Marcus Schäfer <ms at suse.de>
Date:   Mon Mar 18 10:11:41 2013 +0100

    - added option --force-bootstrap in combination with --recycle-root
      in the kiwi modes --prepare and/or --create. When using a base
      tree with --recycle-root kiwi normally skips the bootstrapping
      part because that data should all be part of the base tree. If
      the tree is not complete in this regard, e.g missing zypper one
      can now enforce the bootstrapping phase to be called.
