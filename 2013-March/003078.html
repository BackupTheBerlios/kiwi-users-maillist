On 03/12/2013 02:24 PM, "Horváth Gergely J. (Ottó)" wrote:
> Hello there,
>
> I recently built an other openSUSE image with KIWI and noticed a strange
> thing. I've installed man, but I can not view any man pages:
>
> linux:~ # man man
> No manual entry for man
>
> So I quickly checked if man is installed or what?
>
> linux:~ # zypper if man
> ...
> Installed: Yes
> Status: up-to-date
> Installed Size: 1,8 MiB
> ...
>
> linux:~ # zypper in man
> 'man' is already installed.
> No update candidate for 'man-2.5.2-43.2.2.i586'. The highest available
> version is already installed.
>
> Seems fine. The first idea was to reinstall the package:
>
> linux:~ # zypper in --force man
> ...
>
> And voila, we have man pages:
>
> linux:~ # man man | wc
>      599    4034   30740
>
> Why is that? See the attached config.xml for the image configuration.

You have

<rpm-excludedocs>true</rpm-excludedocs>

in your config.xml file. Many man pages are marked with %doc in the 
package and thus these files will not be installed.

In addition to that we have some strip code in kiwi that removes 
documentation. Thus depending on what's in your config.sh files there 
may be an additional way the man pages that are not marked as doc get 
removed.
