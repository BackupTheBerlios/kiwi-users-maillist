* the missing parport and lp modules problem has been fixed in 2.08
  Anyway these modules are not needed for booting an iso.

* the problem you are facing is very strange because we didn't get
  a size information for /dev/ram1. I saw you ran the image in qemu
  could you test if it works if you call qemu the following:

	mount -t tmpfs -o size=512m none /dev/shm
	qemu -cdrom <your-iso> -m 256

  Additionally could you tell me the contents of your config.xml
  file ?

* if you add the following option at the kernel commandline

	kiwidebug=1

  you will get a shell instead of a reboot exception. When in this
  shell type:

	sfdisk -s /dev/ram1

  and tell me what it prints. Additionally the information of

    cat /etc/modprobe.conf.local
 
  and

    lsmod | grep rd
