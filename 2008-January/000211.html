That means somebody or something has changed data in
/usr/share/kiwi/image/isoboot/suse-10.3 which is evil because the
information there is a kiwi boot image and controls the boot process.
There is no reason why the data there should be changed. Because many
people change something there and wonder if the later system doesn't
boot I have implemented a strict checking here.

To solve the problem re-install the package

	kiwi-desc-isoboot

and call again
