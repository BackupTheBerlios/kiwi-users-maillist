sorry I don't understand. This config.xml is a boot image description
as I wrote in one of my last mails this is used for booting only and shouldn't
be changed. I'm not aware of what YaST2 product creator does exactly but
somehow it must create a system image description and therefore a config.xml
file needs to be created. If I have a look at the file below it looks really
strange:

> <image name="initrd-isoboot-suse-10.3" schemeversion="1.4">

the system image name is like the name of an isoboot boot image. That's
strange and will cause conflicts

>         <preferences>
>                 <type boot="isoboot/suse-10.2" primary="true">iso</type>

the boot attribute contains isoboot/susue-10.2 and will therefore create
a suse-10.2 boot image for booting a 10.3 image. Not a good idea could
work but doesn't have to.

I'm really sorry but I don't think the image creators works
reliably. I have provided example image descriptions with kiwi which you
can find here:

	/usr/share/doc/packages/kiwi/examples

I recommend to start with one of these
