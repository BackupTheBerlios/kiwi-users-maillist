I see your point and would like to suggest the following. The
KIWILinuxRC.sh contains a function named runHook. You can give it
a paramater which is the name of a program which is called. The
program is searched in /hooks and called as /hooks/$program.sh

- If you include appropriate runHook calls into the kiwi
  isoboot/suse-linuxrc file and send that as a patch to the list
  I will check this patch in

- The additional code you need to satisfy the customer should be
  implemented as such hook scripts. You are now able to branch off
  the original isoboot image and still stay with the latest code
  changes we make on kiwi

Would that be ok for you ?
