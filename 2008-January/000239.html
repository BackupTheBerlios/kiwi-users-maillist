Hello, I'm trying to create a vmware image based on kwliveCD.  When I
boot the created image in vmware server 1.0.4 X crashes whenever gdm
tries to start it.  It starts just fine in qemu.

Attached are the X log, conf, and the gdm messages.  They don't appear
to me to be very helpful though.  There's something in the X log that
refers to write-combining and something in dmesg that says:

mtrr: your processor doesn't support write-combining

Just to be sure that it wasn't one of my customizations killing it I
tried the following and got the same result:


kiwi --prepare /usr/share/kiwi/image/kwliveCD-suse-10.3
--add-profile=GNOME --ignore-repos --add-repo
http://download.opensuse.org/distribution/10.3/repo/oss/ --add-repotype
yast2 --add-repo
http://download.opensuse.org/distribution/10.3/repo/non-oss/
--add-repotype yast2 --add-repo
http://download.opensuse.org/update/10.3/ --add-repotype rpm-md

kiwi --type vmx --create /tmp/kiwi.q21170 --destdir /kiwi/images/
--ignore-repos --add-repo
http://download.opensuse.org/distribution/10.3/repo/oss/ --add-repotype
yast2 --add-repo
http://download.opensuse.org/distribution/10.3/repo/non-oss/
--add-repotype yast2 --add-repo
http://download.opensuse.org/update/10.3/ --add-repotype rpm-md

Any help is appreciated.  This is for the Mono vmware image.
