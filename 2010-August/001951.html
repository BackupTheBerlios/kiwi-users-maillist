Am 06.08.2010 um 09:00 schrieb Marcus Schaefer:

>> 1) Is there a way to debug the autoyast profile? It appears as if my
>> autoyast profile isn't being executed on first boot.

you can't overlook that actually. Yast starts during the first boot then and you can see autoyast how it says something like "Configuring Network ...", "Configuring Users ..." and so on.
If YaST does not start on the first boot of your image, autoyast will not configure anything either.
You can also check /var/log/YaST2/y2log at the end to see if autoyast was there.

>> 2) Would autoyast as handled by OEM boot images properly handle
>> pre-scripts?  As far as I can determine, pre-scripts are the only way
>> to dynamically modify the yast profile via a shell script, which is
>> something I need to do.
> 

that stage of the installation is not available when you do image installation so pre-scripts in the XML file will have no effect. chroot-script neither.
The first script you can use of autoyast is a post-script but you cant manipulate the XML file with that.
I think kiwi has an own script too that runs very early and should be useable to manipulate the XML file.
Marcus? Is there a script that runs before yast would start?
