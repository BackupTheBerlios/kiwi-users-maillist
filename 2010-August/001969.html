Changes Report kiwi v4.56 2010-08-08 - 2010-08-13
--------------------------------------------------
most important changes for this weeks kiwi release...

* commit 67e284219dbf8015680877c8501715ba4a4a621a

  fixed permissions and ownership setup of read-write part in
  split images. The problem here was that the read-write part is
  created as new directory structure belonging to root. Therefore
  it's required to update permissions and ownership from the original
  tree afterwards in the same way as it's done for the read-only and
  temporary parts

* commit 25f2369ef8b5660367e7d40851fa7bf6c0d7a1eb

  make use of the --recursive feature of insserv and remove
  my own recursice insserv implementation. kiwi exports a
  function called suseInsertServer() which you can use inside
  the kiwi scripts config.sh / images.sh. The function
  inserts a service with all its dependent services. Current
  insserv has an option for it now which is used now

* commit 4847491e895a4554b90b07d902646fc84bcd0309

  added acpi modules to kiwi's oemboot initrd. This is required
  mostly for fan control on laptops and therefore very important
  for the preload images running on laptops to prevent overheating
  of the devices

* commit f7ff479713ad1a891f1675b9db2b12362a17bac5

  fixed hybrid persistent feature. The check for the bios
  protection was broken which caused the creation of the
  persistent write partition to be skipped.

  added support for the kernel option wipecow=1. This option
  allows to wipe an eventually existing cow file on a hybrid
  iso image to start from scratch. For example if you have
  created a live iso and you want to revert all changes you
  have done so far on the stick just boot with wipecow=1
  and the stick is clean afterwards

* commit 586bf95c469f464433a8f65c9028f518dd768854

  update documentation and man pages. some formal and
  consistency changes over the entire document. Thanks
  to Thomas Schraitle for the good work

* commit c395085300667a0b6dd403c782f43b1ee6cf3e35

  when using the CONF variable along with a pxe image the
  files are not downloaded if the image was installed to
  RAM. This fix makes sure CONF is handled correctly no
  matter where the system image is downloaded into
