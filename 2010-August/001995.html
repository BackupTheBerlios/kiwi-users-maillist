Changes Report kiwi v4.58 2010-08-22 - 2010-08-27
--------------------------------------------------
most important changes for this weeks release...

commit 72010bb6fd4b5cf4c0402e4b042b2c80dfeecffb
commit f692f6bb1feb1b1731f4c51b1f45406eae648ec5

    - added package/group check before yum is called to install
      packages/groups. This is done because there are yum versions
      which silently ignore missing items and proceed installing
      the rest.

commit 88d1939d1a9a13b86daf0bf4f910a0894d718143
commit 6aaec94178eda67078cfd455cc9fe07bfaa61c9b

    - added clicfs for RHEL 5.4 to kiwi rhel repo
    - changed iso image type for RHEL template to use clicfs
      instead of squashfs

commit c451f091a0a60e114c2c224bf516d28d092ee4d6

    - fixed clicfs mount by not using losetup on the clicfs file
      before mounting it via the clicfs command. The clicfs command
      now get the clicfs file directly. Problem here was that with
      a loop clicfs is no longer able to read the size of the clicfs
      file which results in unexpected behavior (bnc #629543)

    - Related to that fix clicfs based caching now works stable
      This means you can call the kiwi prepare step with the
      option:

          --cache /some/directory

      If there is a cache file representing a part of the image
      it will be used and the packagemanager only has to install
      the rest. The cache itself is a clicfs file which is mounted
      as an overlay filesystem with a cow file. This speeds up
      the prepare step a lot and the caches are re-usable for other
      images too. If no cache was found kiwi creates one

commit 067df34571c31dc4cf219542fff45167b7eada93
commit 448c478f095f1f0907e4779c349081d601ea4c7c

    - call dhcpcd with the option --noipv4ll to be able to check the
      return code of dhcpcd and to prevent the interface from an ip address
      if there was a DHCP timeout (bnc #634057)

commit e3a28e42cae74b54f8bfd870a2df2dde5ee57278
commit ef75108a99a9df741aad4e5ba554b4c2d6f285fd

    - added <hwclock> setup to templates. Use utc by default
      This allows to set the hardware clock to either UTC or
      localtime:

      <preferences>
           ...
           <hwclock>utc</hwclock>
      </preferences>

Remember to have fun :-)
