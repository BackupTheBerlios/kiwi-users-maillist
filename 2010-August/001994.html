Hi John,

> We are trying to modify the boot scripts (dump, include) of a SLES 11 SP1 kiwi oem CD install so that it will copy the .gz image file from the install CD onto the newly created system.  We got it to work by using the CDMount command and the using wildcards (*.gz) to copy the files to the installed system after kiwi "dump".
> But we don't want trouble if anyone adds other matching files to the root of the CD.
> We would like to use the $imageName and $imageMD5 variables that are set in the boot scripts, but we get a file not found error.  We checked for the file using [ -e /cdrom/$imageName] after the CDMount call and it fails.  However if we do an "ls /cdrom" after that check we can clearly see the image file is there.  But if we try to copy the files using the $imageName, we get an error:
> /cdrom/myImage.x86_64.gz not found.
> We have moved back to using the wildcard, but would like to use the imageName variable.
> Do you know of any reason this shouldn't work?

sorry no clue about your changes and why it failed but for debugging
I suggest to add some break points at the problematic code:

   systemException "BREAK" "shell"

and boot with the kernel option

   kiwidebug=1

this will give you at least the change to debug a bit around
the problem when it happens
