Hi,

> I have build an unpacked image by kiwi
> eg. kiwi -p /mykiwi -r /tmp/mybuildroot
> 
> If i want to change something in /mykiwi/root

What does "something" mean ? a new package some
configuration changes ? maybe more ?

> I will rebuild an unpacked image, but it takes long time.
> Is there another method can help me to do that quickly?
> Thanks!

You don't have to rebuild the unpacked image. After you
changed something in /tmp/mybuildroot you can call
kiwi --create /tmp/mybuildroot ... again and the resulting
image contains your changes

* maintaining the unpacked image might require an inplace
  repository (for example git) managing those files or
  directories which are about to change to keep track
  of the changes. But that's only a recommendation

* installing or removing packages from the unpacked image
  can be done with kiwi's upgrade mode

  kiwi --upgrade /tmp/mybuildroot --add-package ... --add-package ... \
       -del-package ...

If you want to speedup the --create step you can think of a
destination directory which is a tmpdir it will speedup I/O here
using a parallel gzip like pgzip also speed things up. You can
set the tool with the option --gzip-cmd ... If you prebuild the
boot image (initrd) you can re-use it again which also speed things
up. This is the --prebuiltbootimage option

Making things fast requires some changes in your image creation
environment. We do a lot in Studio here so maybe building it
inside studio is the best solution ?
