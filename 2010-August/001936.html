Hi,

> I am trying to have a license agreement displayed by an oemboot iso image before it  does the copy to the new system.  I see that there seems to be a license function of the boot image scripts (oemboot/SUSE11/root/dump and include), and it looks like it gets called before the "dump" is written to the new system.  Is this meant to display licenses, and ask for acceptance of the license before allowing the image to be written to the new system?  And if so, how do I add licenses to the boot image so it gets accepted before the user installs the system.

create a tarball which includes the license file(s) in the form

  license.txt
  
or language specific

  license.$code.txt

code should be a two letter locale name like "en", "it", etc...

* Once you have that tarball put it into the overlay directory 
  of your image description directory. This is the directory named
  "root/".

* After that add the following line in your config.xml file

  <packages type="image">
     ....add the following line in here
     <archive name="your-tarball-filename" bootinclude="true"/>


the bootinclude attribute is important. If you miss it the
tarball won't be installed in the boot image (oemboot) where
it is required to display the license
