> Hopefully this isn't my fault too :)

no this is mkisofs's fault :) the standard iso-level supports
only a directory depth of 6 but the tree you are about to
store into an iso is deeper. The solution is to pass the
option

    -iso-level 4

to the mkisofs call. kiwi does that automatically if no
compression is used for the root filesystem. I'm wondering
why it is not used in your case and I guess it's just because
your kiwi version is not the latest one. Please check
with

   rpm -q --changelog kiwi

if you can find something like:

   -------------------------------------------------------------------
   Wed Feb 11 12:20:42 CET 2009 - ms at suse.de

   - v3.15
   - use iso-level 4 live CDs if files are stored as
     uncompressed data on the ISO filesystem

if you don't find that changelog entry your kiwi is old :-)
if you find it send the config.xml so that I can try
to reproduce the bug
