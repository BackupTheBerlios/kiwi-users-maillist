Hi,

> Naming looks inadequate, IMO.
> Why not make kernel's type the separate option
> in this context (even dedicated profile might work).
> Otherwise it may end up with all many-to-many
> combinations.
> 
> Let's say
>  bootprofile="diskless,pae"
>  bootprofile="diskless,default"
> etc.,
> or
>  bootprofile="diskless" kernel="pae"
> 
> What do you think?

yes way better. I have named the attribute
 
   bootkernel="pae"

So Roger with version v3.33 you can set

   bootprofile="diskless" bootkernel="pae"

cyberorg: I also make sure you don't need to change your
scripts. You are creating the boot image with --add-profile diskless
and that will work too. kiwi checks that no kernel was added
and adds the standard (default) kernel. What profiles will be
added are listed in an info block named:

    Using profile(s): ...

In your case this looks like

    Using profile(s): diskless,std

ok, hopefully with the next version all people are happy :-))
