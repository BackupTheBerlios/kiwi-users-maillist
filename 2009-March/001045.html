Hi again. 

I'm getting some file ownership problems that seem to be tied into the UIDs 
being different number on the image than it is on the build system. 

I specifically notice this with a gnome image I've produced for a virtual 
machine. 

On the virtual machine I get
# ls -la /var/lib/gdm
total 16
drwxr-x---  3  110  114 4096 2009-03-19 13:35 .
drwxr-xr-x 46 root root 4096 2009-03-19 13:45 ..
drwxr-x---  2  110  114 4096 2009-03-19 12:53 .gconf.mandatory
-rw-r-----  1  110  114  276 2009-02-26 15:47 .gconf.path

i.e. there's no user for those files, 

# grep gdm  /etc/passwd
gdm:x:50:414:Gnome Display Manager daemon:/var/lib/gdm:/bin/false


On the build machine I get 
#  grep gdm  /etc/passwd
gdm:x:110:114:Gnome Display Manager daemon:/var/lib/gdm:/bin/false


so the files have been created with the ownership UIDs for the build system, 
not the image system.
