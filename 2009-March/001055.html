Hi,

> I am having troubles installing a live openSUSE 11.1 on my usb thumb drive. 
> Each time I try, the install works for a day or two then I get major 
> filesystem errors and eventually cause the install to fail to boot.

Hmm, do you use squashfs for the root filesystem ? 

> As soon as I had all the apps installed, the drive started to fail. /usr/bin 
> contained entries from 1969, I saw /usr/bin/sed: Permission Denied when 
> logging in as root, /var/log/messages contained entries that said I was trying 
> to access files outside of the partition (??).
> 
> I have tried both ext3 and ext2. I turned on directory indexing and also 
> turned off file and directory access times (noatime,nodiratime in fstab).
> 
> The USB drive is new. It's a Corsair Flash Survivor 8GB. It's made of 
> "airplane steel". I bought it because it's rugged and should be able to take a 
> beating.
> 
> What special considerations are needed when creating a USB live image?

USB storage is handled in the same way than "standard" hard disks
So I can't think of any special considerations one has to take care for.
We had problems booting from sticks but that always was a Stick/Bios
problem and was not related to the data integrity on the storage
device itself

I didn't had that issue before. Maybe it's the >4GB flash storage
problem discussed on the kernel mailing lists. You can try to add
a fixed system size to your kiwi image description which is below
4GB so that we can ensure the system will not grow beyond this
border

    <type ...>oem</type>
    ...
    <oem-systemsize>3800</oem-systemsize>

Do you see the same problems with this setup ?
