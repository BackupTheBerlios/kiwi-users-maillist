Thanks for your detailed explanations, Marcus,

> kiwi supports an oem-partition-install mode along with

I tried this but it won't work in my usecase,
in particular because I have a very elaborate
partitioning scheme (several lvm volume groups with
preexisting filesystems) and there is almost no disk
space for the creation of a temporary image root;
it has to be written directly into its final place.

I patched some kiwi-modules to
 * permit an existing and non-empty root directory.
Also, due to disk space constraints, I had to
 * add "--download as-needed" to
   the zypper command line of kiwi.
With these settings, the kiwi perpare step is able
to install a working system directly on hard disk.

My question to you, Marcus:
Would you be willing to accept patches
which implement these two functionalities?

My current use-existing-root patch is ugly and breaks
other kiwi functionality, but I can improve it,
possibly by adding a --use-existing-root option
to the kiwi executable.

I attached the patches that I used for testing.
