Hi,

> I'm trying to create an oem (raw) image that I can dd onto a drive:
> 
>     <preferences>
> 	<type primary="true" filesystem="ext3"
> boot="oemboot/suse-SLED10-SP1">oem</type>
> 	<type filesystem="ext3"
> boot="vmxboot/suse-SLED10-SP1">vmx</type>
> 	<version>0.1.3</version>
> 	<size unit="M">512</size>
> 	<packagemanager>smart</packagemanager>
> 	<rpm-check-signatures>False</rpm-check-signatures>
>     </preferences>
> 
> The kiwi --prepare goes fine, but I'm getting errors on the kiwi
> --create because I don't have qemu installed.
> 
> Is qemu a requirement?

yes because I use the qemu-img tool to create/convert the image
format. qemu is required by the packages kiwi-desc-vmxboot and
kiwi-desc-oemboot which you need to have installed for the vmx
and oem image types

> How can I just get a raw image I can dd onto a drive?

oem is the right type for you but you need the tools qemu
provides. A standard raw format can be easily created by a
dd sparse file but I want to make use of all open virtualisation
formats which are tracked very good by the qemu project.
This also includes simple raw formats so I use one tool for
all of them
