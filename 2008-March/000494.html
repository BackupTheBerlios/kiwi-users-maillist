> Ok,

> I need sme hints.

> What is the right way to insert an own software-configuration.
> I m using the test 2 example to build an really small pxe-system that is
> working fine.
> 
> a)
> I want to insert mailx.
> It s an standard suse package.
> Where can I configure this package so that I is installed too after kiwi
> is updated.
> b)
> I want to insert other non-Standard-RPMs. For example packaged from
> packman. The packages are sored local because internet is not avaliable
> while using kiwi. This is not a "kiwi-problem". The whole network has no
> internet-access.

If they are in your regular repository where the openSUSE install came
from, add this to your config.xml:

	<package name="myPackage"/>

If you have a directory with RPMS (I would think any format that smart
likes - but I have used RPMS), add this to config.xml

        <repository type="rpm-dir">
                <source
path="/tmp/RSofT-Mini-openSUSE-10.3/repository"/>
        </repository>

I added this after the entry for the openSUSE repository.

In /tmp/RSofT-Mini-openSUSE-10.3/repository (or whatever path you like)
you put your RPMS. Then select them via the usual <package
name="myPackage"/> syntax.
