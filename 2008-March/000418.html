> I forgot that the initrd is created by the rpm install scripts when the
> kernel is installed.  
> 
> It looks like I need to specify a KIWI_INITRD=initrd in the config.MAC
> file.  Right?

if you specify a KIWI_INITRD and KIWI_KERNEL line in config.MAC
the netboot linuxrc will download the specified files into your
system image and use them for the local grub setup

In any other case the distribution specific tool (on suse mkinitrd)
is used to create the local boot setup. This requires the kernel and
grub to be part of your system image.

The KIWI_INITRD and KIWI_KERNEL information is only needed if the
standard distro tool can't boot your image. In case of a split system
they are required because the suse initrd can't boot such a system.
So yes you need the KIWI_INITRD in your config.MAC
