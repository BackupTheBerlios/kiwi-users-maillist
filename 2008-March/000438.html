Hello,

the current --bootstick option with kiwi only uses grub.
Grub will only work on about 80-90% of all USB sticks since there are 
frequent problems with geometry and BIOS.
syslinux will boot on about 95% since it does not need any geometry (if 
I understood this correctly).
Is there a way to provide a bootstick with a syslinux boot method from kiwi?

I tried to apply syslinux manually (using the initrd and linux built 
with kiwi --create), but the ancient syslinux 3.31 coming with openSuse 
caused problems and tried to load "syslinu.x" instead of "linux" because 
linux is in the dir "syslinux". I had this problem before on my Fedora 
builds, and the solution was to update to 3.51 since 3.31 doesn't 
recognize FAT directories. But syslinux 3.5x won't build on my opensuse, 
I did not find out the reason yet, and binary packages are not available 
from the factory. How can this work for live CD/DVDs using kiwi if the 
linux is not living in the root dir of the CD/DVD?

Bernd
