There is a method to circumvent this.
To be able to use the same stick for multiple machines (of the same arch, or a 
32 bit image which works on x86_64 at least), you can pass a 
parameter "RECONFIGURE" to the init script.

It is then passed to /etc/init.d/create_xconf

If you used the kwliveCD-suse-11.0 image description
(https://forgesvn1.novell.com/svn/opensuse/trunk/distribution/images/kwliveCD-suse-11.0)  
you already have this feature.
If not, you can patch ./root/etc/init.d/create-xconf in your image description 
directory.

Maybe this approach solves your problem?
