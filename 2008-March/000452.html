Another question has occured to me today when dealing with the kiwi 
sticks on different machines.
Sax autoconfiguration is taking place on initial startup of a kiwi USB 
stick, and xorg.conf is written to the persistent partition.
If the same stick is booted next time on another PC (which is quite it's 
purpose), xorg usually fails because the old xorg.conf with the prior 
configuration of graphics card, mouse and monitor is used.
Shouldn't there be some instance in opensuse that autodetects the 
hardware configuration and - in case there are some differences - 
reconfigures the system or X11?
In Fedora/Red Hat this is usually done with kudzu, but I didn't find 
anything like that in openSuse except yast and sax.
