Hmm, you should try to find more information about the situation
before the du call. To be able to do this do the following:

	1) boot with the kernel option:

         kiwidebug=1

    2) switch to the logging console Alt-F3 as soon as you can
       see that kiwi has enabled logging on this terminal. This allows
       you to to scroll the history

    3) wait until the systemException happens. As you are on the
       Alt-F3 console you can scroll with Shift+Picture Up/Down
       investigate if you can see other errors
