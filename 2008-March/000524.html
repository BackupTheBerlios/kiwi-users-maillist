> I have installed KIWI-LTSP on openSUSE 10.3 x86_64. When I try to boot first 
> thin client, it displays an information:
> 
> Creating device nodes with udev
> Boot logging started on /dev/tty1 (/dev/console) at Tue Mar 25 12:10:50 2008
> -----> Starting boot shell on tty2
> -----> Boot-Logging enabled on /dev/tty3
> -----> Kernel logging enabled on: /dev/tty4
> -----> Network module: Failed to load network module!
> -----> rebootException: error consoles at Alt-F3/F4
> -----> rebootException: reboot in 120 sec...
> 
> Error console shows that client tries to load 8139too module. Thin client has 
> Realtek 8139 based NIC.

if you boot with the kernel parameter

	kiwidebug=1

you will get a shell instead of a reboot exception. The kernel parameter
in case of a PXE boot needs to be setup in the pxe configuration file
on the boot server

the shell allows you to investigate the problem more detailed.
If you got the shell call:

	hwinfo --netcard

and provide the information displayed.

The information on the kernel console Alt-F4 should give us more
information why the module loading has failed. I think the 8139too
module is the correct one
