> Another question has occured to me today when dealing with the kiwi sticks 
> on different machines.
> Sax autoconfiguration is taking place on initial startup of a kiwi USB 
> stick, and xorg.conf is written to the persistent partition.
> If the same stick is booted next time on another PC (which is quite it's 
> purpose), xorg usually fails because the old xorg.conf with the prior 
> configuration of graphics card, mouse and monitor is used.
> Shouldn't there be some instance in opensuse that autodetects the hardware 
> configuration and - in case there are some differences - reconfigures the 
> system or X11?

you are definitely right here and yast provides a basic system to
handle that but we have a hard time integrating it and I would prefer
a solution which works without yast. I think it would be a nice task
for someone who is interested to implement such a mechanism which checks
at boot time if the hwinfo provided hardware ID's has changed. This is
also an area where I didn't found time to work on yet
