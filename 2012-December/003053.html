last time i tried zypper --root it would complain about some
users/groups it wanted and my host system didn't have ( i was using
zypper from my gentoo installation ) - this could lead to some runtime
problems. So I would not recommend it, unless you run it from suse
system itself.

As for kiwi, you will probably need to hack kiwi's scripts to work
around that block ( potential location would be createDirInteractive
in /usr/share/kiwi/modules/KIWIGlobals.pm )

Or you could build a tarball with kiwi and unpack it into target
location. This is what i do, and i couple it with a custom
installation script that sets up my storage accordingly, unpacks the
tarball and installs the bootloader. Of course using it for a one-time
installation is a bit of an overkill.
