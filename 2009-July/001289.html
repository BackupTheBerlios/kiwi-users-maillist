Hmm, no I think the fallback code when no CD drive was found applied
in your case. I think there is some other bug maybe related to 11.2
Would you mind to do the following:

   - boot with: kiwidebug=1
   - when the systemException happens you will get a shell
   - store the file /var/log/boot.kiwi
     ( I always do that with an USB stick which I plug in _after_
       the exception happened to have a storage device for data )
   - send the log

> I could resolve the problem by attach my USB disk and run the livecd
> iso as CDROM (I'm using VirtualBox 3.0.2 as system test).

Oh wait does that mean in your first attempt the live CD was not
used as CDROM device ? If so it's important that the virtualization
layer uses the live iso file as cdrom source because the kiwi initrd
for a liveCD of course expects a CD/DVD device which it also searches.
When I test a live ISO in qemu I always call it with

   qemu -cdrom /path/to/iso

could you test if the iso works in that case ? you also can
install kvm and run qemu-kvm it will speed up things a lot :-)
