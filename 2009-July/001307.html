> Is this possible to completely running openSUSE from RAM (using RAMFS
> or TMPFS) which should be faster than a standard disk? I would like to
> test openSUSE with standard app (office with kde or gnome base or even
> with xfce or icewm).
> 
> Our workstation has minimum 4-8 GB of RAM and all of data load and
> save onto server. I know it's possible to do that with small Linux
> distro like Puppy or DSL but it should be an interesting work to make
> openSUSE run on RAM instead of disk. If a small size is the key, I
> want to test it with limejeos package.

kiwi supports running the OS in RAM only via the PXE boot
process.

The kiwi initrd loads the kernel and initrd via PXE/TFTP
from a boot server. When the kernel has unpacked the initrd 
the initrd code will search for a configuration file which tells
the system where to obtain; and how to activate the root filesystem.
One of the supported modes here is to load the root filesystem image
into the RAM of the system.

The root filesystem image can be stored in any location supported
by curl or atftp which are the tools used to download the image.
At the end your OS runs in RAM

You can ask the LTSP people on the list here for the setup
of the boot server. They know much about that setup
