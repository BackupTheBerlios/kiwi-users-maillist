Basically this problem will go away as soon as X doesn't require
a configuration anymore. I have CC'ed Stefan Dirsch maybe he can
say some words about that. It looks like 11.2 will be the first
distro without sax2

for the older distributions you always have to run sax2
if the machine changes (basically also if the graphics card changes)
You can think of a system which creates an md5sum from the output
of

  1) the output of dmidecode
  2) the output of sax2 -p 

if the md5sum doesn't match on boot something has changed which
surely requires a new X configuration. sax2 -r -a should be
able to create a new default setup then. You also can try to
work with X -configure but that will not work for binary
drivers
