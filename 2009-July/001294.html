>> It applies to all image types. In case of a virtual disk creation the
>> size references the disk size and in any other case the size references
>> the size of the root filesystem
>>   
> If I understand this correctly this will create a big problem for me. I 
> have a description that is used to create an OEM image and a VMware image. 
> Thus if I set size to 50G to get a 50 GB VM disk I also only get a 50 GB 
> root partition in the OEM image. However, in the OEM image I want the root 
> partition to use the available space on the hard drive (minus the 128 GB 
> swap) I've set up.

I don't think so. If you set <size unit="G">50</size> the result is
a 50GB virtual disk for oem and vmx that's correct. But this size
setup has no impact on the repartitioning done by the oem initrd on
first boot. It still will use the available space on the hard drive
as you configured it with the oem- options. Which oem- Options have
you set ?
