> > I think the problem is that one or two of the sample templates you're
> > using have a predefined kdm as the login manager. So when you select
> > gnome, it is still trying to use kdm and defaults to xdm or whatever.
> >
> > check the config/etc/sysconfig directory
>

I had the wrong directory. 
Taking the example of the suse-oem-preload the file the gdm definition is in 
is. 
/usr/share/doc/packages/kiwi/examples/suse-11.1/suse-oem-
preload/root/etc/sysconfig/displaymanager

it has an entry DISPLAYMANGER="kde" there, which as kde isn't installed, falls 
back to gdm. 

very much a "doh !!" moment when I noticed, especially as I'd already coded a 
script to change it back to gdm. :)
