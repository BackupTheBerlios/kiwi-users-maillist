> Thanks, does this apply to other images types in some way? If it only 
> applies to VM (Xen and VMware) images I wonder if it shouldn't be moved to 
> the appropriat options for those images.

It applies to all image types. In case of a virtual disk creation the
size references the disk size and in any other case the size references
the size of the root filesystem

> Further I noticed that all the oem- options are part of preferences and was 
> thinking it would be nice to collect these in an oem-options element.

Well I agree putting them all into the preferences section was not
the best place. If we change this we also should think of changing the
type schema completely. I was thinking on something like this for each
type:

   <oem attributes>
      <size .../>
      <oem-parameters/>
   </oem>

   <vmx attributes>
   </vmx>

   <iso attributes>
   </iso>

   ... and so on


This would be a hard change and we need a very good xsl stylesheet
to transform from the old scheme into the new scheme. I have that
on my todo list but prio is low atm
