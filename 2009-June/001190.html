IMHO the -info <something> interface is a good option. One option should
be -info all that creates an aggregation off all the subcategories.

One thing I am struggling with conceptually is the duality between what
is in the image description and what is available. The original question
was directed toward availability of patterns and not the patterns
already described in the image.

For example, if we use

# kiwi -info patterns

does kiwi return a list of patterns described in the image or does kiwi
return a list of available patterns in the configured repository? What
if I want kiwi to be able to give me both answers?

# kiwi -info-repo patterns
# kwi -info patterns

One option for information from the repository and the other for the
image description?

Is this a scalable approach and applicable for other options?

As far as the image description is concerned I think it would be nice to
get a list of the configured image types (image info only), a list of
overlay files (image info only), a list of packages (available in repo
[repo info], to be installed [image info]), and the boot image to be
used (image info only).

I think there are two problems to be solved.

Thanks,
Robert
