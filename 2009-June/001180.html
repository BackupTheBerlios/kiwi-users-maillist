Hi,

> Hi all, 
> 
> Does the --migrate option work with sled11 ? Or is there any plans to make it 
> work ? 

I didn't have much time to work on migrate so I wouldn't rely on
that feature at the moment. There are nobody requesting it for a real
business case which is the reason why it is a playground only atm.
If there is someone willing to take over that part of kiwi and
officially support it I would have no objections

The migration table: modules/KIWIMigrate.txt currently did not
include repositories for SLE11 so the migration surely will not
recognize SLE11 at the moment. This will not prevent the migration
to run but it will leave you with some details you need to add
manually afterwards

The design of the migration is also based on an inspection of the
system and the result is turned into a kiwi image description.
I myself doubt that the result can be a 100% identical clone
represented as a kiwi image description. So I think there always
will be some manual work after the migration is done. The code
which inspects the system of course could be improved and be
much more intelligent as it is right now.

I think the feature would be worth to become supported but
it must be clear to the user that the result needs a human
validation afterwards.
