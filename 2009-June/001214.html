Hi Robert,

> How can one consistently determine whet package names are expected?
> 
> For example, trying to create an image that contains ImageMagic requires
> the following:
> 
> <package name="ImageMagick-6.4.3.6-5.3"/>
> 
> just using
> 
> <package name="ImageMagick"/>
> 
> triggers a "package not found" error. Yet, with GIMP one can get away with:

I can hardly believe that. If you call

   rpm -q ImageMagick --qf %{NAME}

you will get the name which can be resolved by the packagemanager.
In case of ImageMagick the result is ImageMagick. So I think some
other error causes the "not found" problem. Are you really sure
the ImageMagick package exists in the desired arch on the repo(s)
you specified for building ?
