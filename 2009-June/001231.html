Hi,

There is an "opensusePattern" element, but I fail to see one for 
SLE{S,D}. AFAIK patterns are supported on SLE{S,D} and it would be nice 
to be able to use them for images created by Kiwi based on SLE{S,D}.

An example where the use of the "opensusePattern" pattern breaks down 
when using it with SLES10 SP2 is "gnome". The "gnome" pattern in 
openSUSE pulls in Evolution. For SLES Evolution is not part of the 
distribution, thus I presume the SLES "gnome" pattern does not have this 
dependency.

Thus I either have to specify all the packages I would want from GNOME 
in config.xml or I have to use the "opensusePattern" and use both the 
SLES and SLED ISOs to use "gnome" as a pattern.

It would be nice if this would be more generic with a type specifier, 
something like this maybe:

<pattern name="gnome" type="SLES"/>

supported values for the type attribute could be SLES, SLED, openSUSE.

Unless this is an underlying issue with the pattern definition in the 
various flavors. However, even if this is the case using 
"opensusePattern" as an element type is probably a bit to specific for a 
tool striving in some ways for distribution independence.
