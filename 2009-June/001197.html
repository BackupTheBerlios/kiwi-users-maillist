Yes, that's what I was afraid you were going to say...

I looked at the src rpm for packaging a kernel, and it seemed quite
daunting, due to the post-processing.   Building and installing a
kernel on my local system, I can do.  Packaging one up seems like
it'll take a bit more effort, unless I make the package build from
source.

I was hoping there might be some way to run the kernel "make install"
step on the liveiso image without packaging the kernel.

My primary need is that I needed a 2.6.24 or newer kernel to support
my laptop hardware, and the newest one I could pull in for suse-11.1
was 2.6.23.

I tried specifying some priorities on the repos using the smart
manager and the factory base repo, but that didn't seem to work
either.

What I ended up doing over the weekend was building a hybrid 11.1/11.2
system since my 11.2 install dvd iso ships with a 2.6.30 RC kernel
snapshot.   I used the 11.2 baseline, then I pointed to the 11.1
packman and opensuse.org repositories.     That got it working so long
as I booted it in failsafe mode -- I'm still debugging the hang after
login in the non-failsafe mode.

One thing I haven't tried is using the
http://download.opensuse.org/factory repo instead of the other distros
as a base.  Maybe that would be a better strategy?
