Hi,

> The next issue I hit, which has haunted me for two or three weeks, is
> that ConsoleKit-32bit is a requirement, but is not a resolvable
> requirement, if you use http://download.opensuse.org for your primary
> 11.1 repository.   I have no idea why the ConsoleKit-32bit package no
> longer exists in any reachable repository.   I solved this issue by
> using an openSUSE 11.1 installation dvd as the primary repository
> rather than htt://download.opensuse.org.

*whops* that sounds really strange to me. ConsoleKit-32bit exists only
in the 64bit tree and should not be required by the packagemanager
if you build an image for 32bit. As you prefix all kiwi calls
with linux32 this problem should not happen.

Have you tried to remove the cache files before calling kiwi:
( I assume you use zypper to build the image)

  rm -f /var/cache/kiwi/zypper/*
  rm -f /var/cache/kiwi/zypp.conf*

  linux32 kiwi --prepare ...
  linux32 kiwi --create ...

There should be no reference to ConsoleKit-32bit. If yes send the
kiwi logs for this call.

Thanks for you feedback
