Hi,

> I have a system that uses the Intel e100 driver. The last KIWI image I
> made for this type of system was with openSUSE 10.3, when, it seems, the
> e100 driver did not load any firmware. The e100 driver in openSUSE 11.2,
> however, does require firmware. I see in the failed boot that the
> firmware file was not found, and it is indeed not present.

We had a similar problem with an adaptec driver. And I recently added
at least that firmware to be present in the initrd. It's this kiwi
changelog entry:

-------------------------------------------------------------------
Thu Dec 17 12:04:12 CET 2009 - ms at suse.de

- removed adaptec-firmware from SLED boot images. The package
  does not exist on SLED (bnc #562224)

With this change the /lib/firmware directory will not be removed
from the initrd as it was before. So you have a good chance if you
add the eventual missing firmware package that it stays inside the
initrd :-)

> I see that the root file that I eventually mount (nbd or aoe) does
> contain all the firmware in /lib/firmware/<kervelVersion>. However, the
> initrd does not contain /lib/firmware at all. And it is then that this
> is needed, as that is when the e100 driver is loaded: before the main
> image is mounted.

yep this is fixed if you use the kiwi version with the fix mentioned
above. The initrd's created now have all kernel installed firmware's
on board
