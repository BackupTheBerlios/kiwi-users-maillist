I have updated my kiwi to 3.92-80.1 to get a correction in the
way /lib/firmware is handled. I am happy that this is now as needed.
However, the boot now has a place where it seems to wait a bit. 

On the console. It prints:

Set System Time
registerd rtc_cmos as rtc0

and then waits 4-5 minutes. When it starts up again, it prints so fast
that I cannot see what it writes. The boot continues to a successful
point, afaict.

On the virtual terminal 3, I see:

exec chroot . /sbin/bash -c '/preinit ; . /include ; cleanImage ; exec /sbin/init '
read fbnum fbtype
[' '!' -z '' ']
[' '!' -z '' ']
[' clean = clean ']
[' -x /bin/rpm ']
[' '!' -z '' ']
[' '!' -z '' ']
[' '!' -z '' ']
[' '!' -z '' ']
[' '!' -z '' ']

This is also what is at the end of boot.kiwi. Then it just waits.

The console on VT2 shows a prompt, but it does not accept any input.

Eventually (after a couple of minutes), the boot continues to a
successful point, afaict.

This did not happen previous to this release of KIWI. It does this at
the same place each time. I would provide more information, but I am not
sure what more there is to say.
