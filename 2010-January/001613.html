Hi,

> My config.default file (made by hand) contains only:
> AOEROOT=/dev/etherd/e0.1
> This is read, and the root over AoE happens.

ok, so why do you have a pxedeploy section which says something
completely different ?

> But this second config is not seen by my kiwi images. Only the one
> containing AOEROOT.

So you don't use it on your pxe boot server. In that case
simply remove the pxedeploy section... you don't need it as
you maintain your config.default by hand

> Perhaps this explains why I seem to have trouble with making kiwi
> images.

Surely not, image creation is one part your deployment setup on your
pxe boot server is something different

> You mention pxedeploy. Is that different from deploy? 

we renamed deploy to pxedeploy, don't you remember the discussion ?

> How does the IMAGE statement relate to the AOEROOT statement? I would
> think that the IMAGE statement is not used.

There is no relation. If you use AoE you only need AOEROOT and
nothing more. IMAGE is only required if a download to the client
happens. It tells the system the name of file to download and from
where etc...

> Are the DISK and PART items useful when there is no physical disk? Only
> the root file system (made by kiwi) accessed over AoE.

not useful if there is no local storage
