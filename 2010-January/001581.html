Hi,

> thanks for your answer. I use the vmx image as a raw image of fixed size and
> copy it with dd to a disk.
> 
> The oem type is in my understanding a installation system (it resizes itself
> or copys itself to the target disk). I don't want my system to behave this
> way.
> 
> Is it possible to deactivate the installation functionality from the oem
> type?

an oem is also a raw image but with the capability to adapt itself to
a new disk geomtry. It doesn't matter if this disk is a real disk or a
virtual disk. The additional creation of an installation image containing
the oem can be easily switched off. Simply don't specify any format 
attribute in your type section.

    <type filesystem="ext3" boot="oemboot/suse-11.2">oem</type>

this will also create a .raw disk file which you can dump with dd as
you did before. On first boot of that image it will repartition itself
according to your setup. During that process also the swap partition
is created and activated

The initial deployment takes a bit longer compared to a standard vmx
because of that repartition stuff. I hope this is not a problem in
your case
