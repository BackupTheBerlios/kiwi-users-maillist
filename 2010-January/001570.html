Hi,

> I use kiwi to  create images for different targets (usbstick, oemboot, vmx).
> 
> Until now I was able to use a single config.xml. The only problem I have is
> that some image types need to be of a fixed size and others not.
> 
> What is the best way to make the image size dependant on the image type? Is
> it possible to give the image size as a command line parameter to kiwi or to
> (mis)use the profile mechanism?
> 
> Is it possible to use the inheritance mechanism to keep a base config and
> just set the image size in the child-configs?

- you can pass the option --bootvm-disksize when calling kiwi --create
  this should be preferred over the config.xml information

- you can also make profiled <preferences> sections where you can add
  different <size> sections. An example (not with size but with type) can
  be found in the kiwi-templates packages
  (/usr/share/kiwi/image/suse-11.2-JeOS/config.xml)
  But all kinds of odd things could be created that way too :-)

- inheritance should also work but is very untested since nobody really
  rely on it at the moment
