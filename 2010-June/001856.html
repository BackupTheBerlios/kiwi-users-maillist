Changes Report kiwi v4.43 2010-06-20 - 2010-06-25
--------------------------------------------------
most important changes for this weeks release...

GIT: fixed copy of root tree into final image. We used
     cp -a $source/* $dest which did not match the dot files
     in $source/.* Therefore I changed this to use a tar
     based copy

if there are hidden overlay files in / they don't end up
in the final image. This is fixed with this patch

GIT: added oem-inplace-recovery option which allows creation
     of the recovery tarball at first deployment instead of
     creating the tarball and store it inside the image

if you use an oem image with recovery support enabled the
recovery tarball normally is stored in the image itself
and is moved into the recovery partition on first deployment.
With the option above one can request the creation of this
recovery tarball at deployment time instead of creating
it at build time. This reduces the size of the image but
also increases the initial deployment time

GIT: use cylinder based alignment with new parted version
     2.2.x. This fixes the oem repartition code for systems
     which uses parted >= 2.2.x build on a parted 1.9.x build
     system. Also removed unused functions: partedGetSectors
     and partedStartCylinder

when building an image with old parted (on suse 11.2) for
a newer distro like 11.3 the repartitioning will destroy
the data inside the partitions. This fix can handle that
case. What's still a problem is creating an image for an
older distro with new parted on 11.3 for example. Still
searching for a solution here

GIT: fixed <image name="..."/> attribute validation, space
     and / is not allowed for this attribute

It's not allowed to have spaces in name. this fix checks
that at the early XML validation time

GIT:  started support for s390 imaging:
      * zipl support in kiwi. This is still work in progress only
        the skeleton at the required code points has been added.
      * adapted the JeOS sle11 template to provided a profile
        for x86 and s390 packages
      * also added vmxboot and oemboot bootimages to be build
        The code there will work as initrd for an s390 guest
        within kvm

We would like to support the creation of guest images for
kvm on s390. I started to add the relevant code paths which
needs to change but the implementation is not ready to use
so far 

GIT:  fixed split section evaluation (bnc #615121)

if the split type is used there was an error if one excludes
a temporary path which is reserved for later use as a persistent
path 

Remember to have fun :)

Wish you all a great weekend
