Changes Report kiwi v4.39 2010-05-30 - 2010-06-04
--------------------------------------------------
most important changes for this weeks release...

GIT:  removed dbsulock from kiwi, hal is deprecated

 dbus locking was used for --bootstick deployment to avoid
 the automounter from mounting the newly partitioned stick

GIT:  fixed CDMountOption() to use blkid type check (bnc #610649)

 This fix is important of you use OEM installation media
 which is bigger than 4GB. In this case kiwi creates an iso
 with the udf filesystem and therefore the CD mount is not
 allowed to use the -t iso9660 mount option

GIT:  use vmxboot linuxrc for usbboot too, make sure --bootstick
      deployment does a cleanup of the write partition if used

 due to the last refactoring of vmxboot I could safely use the
 same code for usbboot too. In the past I planed to remove usbboot
 completely as we have several other alternatives but as it is
 used actively by the community the code will stay alive

GIT:  use "kernel chain" syntax for syslinux chain loader

 chain loading of the recovery partition if set in an oem setup
 works now with extlinux as bootloader too. The only remaining
 problem is the graphics boot code which breaks the chainloading
 At the moment you need to leave the graphics mode with Escape
 and select the "Revovery" entry from the text menu. Steffen
 Winterfeldt is already looking at the graphics side and it should
 be fixed for 11.3 already

GIT:  don't touch modules in updates and weak-updates (bnc #609879)

 This fix is important for driver updates. For example IBM
 provide an update RPM for one of their drivers and put them into
 the /lib/modules/.../updates or .../weak-updates directory. If
 this package is set as a bootinclude="true" package the driver
 was still removed by the suseStripInitrd() function. The patch
 changed this to not touch any kernel updates directory

Remember to have fun :-)
