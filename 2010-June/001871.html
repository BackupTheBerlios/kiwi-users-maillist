On Wed, Jun 30, 2010 at 04:26:57PM +0200, Marcus Schäfer wrote:
> So do you have a network of clients running managed by puppetmaster
> which allows you to test the kiwi2puppet ? I'm thinking how I could
> take a look into this without too much effort on the infrastructure

No, I have tested it by running "puppet --noop foo.pp" manually from
the console. (Apart from syntax checks of the manifests)

I don't think a master+puppets network is necessary or useful for
this, I am thinking more about a script that would get a machine and
its kiwi description. It would install puppet on it, convert the
kiwi config to a mainfest, log in, run the test as above and collect
the log.

The machine could be specified by network address, or an image to be
run in a VM, or it even could be built by kiwi. Or the input would
be a SUSE Studio Appliance URL and it would download a build and
export its kiwi config.

As for other testing, do you have some collection of juicy
config.xml's to exercise the front end of kiwi2puppet?
Is there a way to query the openSUSE Build Service for packages that
are in fact images with kiwi configs?
