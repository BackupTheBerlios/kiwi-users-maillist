> the only reasonable snippet on it i found said :
> 
> "compressed - Compressed filesystem with squashfs mounted with an aufs
> based overlay mount to allow read-write access.
> unified - Compressed filesystem with squashfs mounted with an aufs
> based overlay mount to allow read-write access."

I explained the flags in the kiwi-doc package or online here:
http://isny.homelinux.com/data/kiwi.html#sec.iso.flavours

> However, when i created an image with such settings
> 
>  <type primary="true" boot="isoboot/suse-11.4" bootprofile="default"
> bootkernel="std" flags="compressed" hybrid="true"
> hybridpersistent="false" image="iso>
> 
> the /root contents are read-only. I actually get the message "read
> only filesystem".

If you specify "compressed" you also should specify which parts
of the filesystem should be read-write. That can be done by
the so called split section. See the split mode docs on the
same page mentioned above

> Does that mean i should go with "unified"?

In a unified setup aufs is used. aufs allows write permissions
over the entire root tree. Problem is aufs is only supported
on SuSE 11.1, before it did not exist and after 11.1 we don't
support it anymore because it is not accepted by the kernel
people

If you want write permissions over the entire tree you should
use clicfs

  <type image="iso" boot="isoboot/suse-11.4" flags="clic" hybrid="true"/>
