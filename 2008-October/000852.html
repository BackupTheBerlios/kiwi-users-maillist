> At the moment the inst-source support in kiwi is used only internally.
> I think, and Jan correct me if I'm wrong, it can't be used outside
> the Novell/SUSE infrastructure at the moment. Of course this is planned
> to be possible in the future

Well, ... there are two minor issues. First, the signing doesn't work 
externally. You can theoretically live with an unsigned repository if you 
know what you're doing or attacks are prevented somehow; you only have to 
specify "insecure=1" as kernel param on boot for the installation system 
kernel.

The other thing is the missing package description (which comes from package 
database) which is used during installation. If those are missing, as far as 
I know the worst that can happen is that YaST can't show the descriptions 
when installing. Once again, if you know what you're doing... ;)

So basically instsource _should_ work outside as well, if you can deal with 
those issues. I admit I never really verified the package description issue, 
but I can do a test run or two on that to clarify the consequences.
