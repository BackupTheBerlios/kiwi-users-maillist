> fix this or trace it in more detail? Any help is appreciated.

> Hmm, looks like a stick / grub problem. Could you try another stick ?
> Alternatively you can try to build a hybrid ISO. That's an iso image with
> a partition table so that the BIOS see this as a disk. As result you
> can dump the iso on your stick and use isolinux (instead of grub) to
> boot that thing. Sometimes this solves the boot problems on the stick
> which is often a combination of grub + bios + bad-stick-hardware
> 
> To build the hybrid use the suse-11.2/suse-live-iso example and
> edit the config.xml in there. Change the type line to look like
> this:
> 
>   
>      image="iso" primary="true" boot="isoboot/suse-11.2" flags="clic"
>      hybrid="true" hybridpersistent="true"
>    />
> 
> You can install the generated ISO file on the stick via dd and
> boot from the stick


That's what I was afraid of. Unfortunately I have no other stick to try it on to verify this. 
Anyway, your suggestion for the hybrid setup worked just fine! Thanks for the advice!

Some notes about that. I have kiwi v3.74 and the schema parser didn't like the hybridpersistent="true" and image="iso" attributes. I was able to run it only like this 
<type primary="true" hybrid="true" boot="isoboot/suse-11.2" flags="clic">iso</type>

All other combinations of attributes and values(!= iso) either didn't parse or give an error on --create:
Use of uninitialized value $type{"filesystem"} in split at /usr/sbin/kiwi line 2273.

Basically, what worked for me is what you have described a while back in http://lizards.opensuse.org/2009/08/05/hybrid-live-systems/
