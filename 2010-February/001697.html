> I am trying to build a liveCD with KIWI that has a web application that is optimized for 1024x768.
> I want the liveCD to 'discover' the best screen resolution, for that I'm using the "sax2 -c 0 -a -i" command, but I'm not sure neither if it's the best way nor if it's the correct tool to do that.
> Any ideas?

on SuSE sax2's auto mode is the best way to do it yes.
But for >= 11.2 there is no sax2 anymore.

Starting with 11.2 sax2 is no longer supported because all X11
configuration tasks are done by the X-Server itself. In that
case simply starting the X-Server will switch to the best
resolution automatically.

If your monitor reports a bigger resolution than 1024x768 as
the best choice the X-Server will start in that resolution.
If that is not ok for your web-app you need to tell the
X-Server that you want to switch to another resolution. You
can use the randr extension to do this. There are several
tools for the different desktop environments which allows
the mode switch. Unfortunately I don't know how one must
provide the information to for example KDE in order to
force 1024x768 on startup of kwm.
