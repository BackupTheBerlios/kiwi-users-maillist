> GRUB freezes at boot time after loading initrd. I have tried booting manually through the GRUB prompt, using the parameters from the generated menu.lst file and:
> - kernel loads fine
> - initrd loads fine
> - when I issue "boot" it freezes. Doesn't proceed to kernel extraction, no output, no usb access, nothing.
> 
> I have tried the oem approach with the same problem. Any suggestions how to fix this or trace it in more detail? Any help is appreciated.

Hmm, looks like a stick / grub problem. Could you try another stick ?
Alternatively you can try to build a hybrid ISO. That's an iso image with
a partition table so that the BIOS see this as a disk. As result you
can dump the iso on your stick and use isolinux (instead of grub) to
boot that thing. Sometimes this solves the boot problems on the stick
which is often a combination of grub + bios + bad-stick-hardware

To build the hybrid use the suse-11.2/suse-live-iso example and
edit the config.xml in there. Change the type line to look like
this:

   <type
     image="iso" primary="true" boot="isoboot/suse-11.2" flags="clic"
     hybrid="true" hybridpersistent="true"
   />

You can install the generated ISO file on the stick via dd and
boot from the stick
