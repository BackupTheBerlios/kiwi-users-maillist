Hi Hans-Peter,

I have tested NBD together with aufs for a base 11.1
root system which was:

  a) uncompressed (ext3)
  b) uncompressed (ext3) / write NBD device was ext2
  c) compressed squashfs / write NBD device was ext2
  d) compressed clicfs / write NBD device was a raw file

all cases worked for me. I also activated NBD swap without
any problems. The config.<MAC> contents was

  a) NBDROOT=192.168.100.2;2000;/dev/nbd0;9120;/dev/nbd2

  b) NBDROOT=192.168.100.2;2000;/dev/nbd0;9120;/dev/nbd2;2001;/dev/nbd1
     UNIONFS_CONFIG=/dev/nbd1,/dev/nbd0,aufs

  c)+d) NBDROOT=192.168.100.2;2000;/dev/nbd0;9120;/dev/nbd2;2001;/dev/nbd1
   (union config is built automatically if the root is a read-only fs)

So NFS sucks for my tests but NBD worked fine, AoE also sucks
the kernel crashed in the same way compared to NFS
