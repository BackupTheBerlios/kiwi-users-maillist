Hi,

> I see no direct config options to do this.  Is there a way to tweak
> things to do this?  E.g. I have a pre-partitioned, formatted, disk (e.g.
> boot/swap/data partitions, etc) and want to dump the image onto the boot
> partition.

If you select a type like this:

   <type image="ext3"/>

the result will be a filesystem image containing only data.
You could dump this kind of image onto a partition. For example
/dev/sda2 is the partition we want to use:

   dd if=image of=/dev/sda2

You might want to resize the filesystem to the current size of
the partition though:

   resize2fs /dev/sda2

> This brings up another question.  How do I control the partitions
> created/used.  E.g. If I have to do a full "install" then rip the boot
> partition off, how do I control the specific partitioning/setup I need?
> There are just some specific root/home/swap eom-* options but how do I
> specify a completely different partition layout?

You can't. With the meaning of an image you can only influence some
basic parts of the partition table like "with home partition or not",
"how big should the system partition be", "with swap or not", "how big should
swap be", etc. For a more detailed separation into partitions kiwi supports
LVM where you can specify volumes with the <lvmvolumes> section.

If that doesn't solve your needs you should consider to use autoYaST
which is not an image based approach but an automatic installation
workflow which allows you to preconfigure all steps of a standard
installation. Within that workflow you also can setup your own
partition table.
