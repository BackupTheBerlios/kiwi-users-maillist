Hi All,

I'm hoping you all had a good start into 2009 and wish you the
best for this new year. For those of you who are interested in
using logical volumes instead of unflexible partition tables
the next kiwi version will come up with LVM support. All you need
to do to activate it and play with it is add the option

    kiwi --create ... --lvm

as part of the kiwi create step.

* Bootloaders can't access the volume group so kiwi will setup one
  small partition with an ext2 filesystem for booting the system.
  Inside the system /boot is referenced to this partition

* Anything else is part of the volume group and can be managed
  by the image user even after the system has booted. Especially
  OEM vendors with special wishes can include them now as part
  of their system image description without the need for an own
  boot image code (initrd) 

* LVM support is available for the image types oem, usb and vmx
  it doesn't make sense for iso and xen. Concerning the pxe type
  I'm not sure whether LVM would make sense or not. At the moment
  I skipped supporting it but I'm open for any suggestions

First kiwi version with LVM support will be 3.03
I plan to release on Friday this week
