Hi,

> Hi, is there any complete tutorial for making oem boot cd with disk
> autopartitioning for headless instalation? Thanks in advance!

I think there is no complete tutorial about this topic. I have
mentioned the possible OEM parameters as part of the kiwi cookbook
(kiwi.pdf) in the kiwi-doc package. The list of possible oem-*
parameters in config.xml is useful

Here is a workflow I use when creating such an install CD

1) I often use a 4GB USB stick just for testing the bare
   system image in the first place.

2) My image description includes among other data
   the following type:

   <type filesystem="ext3" boot="oemboot/suse-11.1">oem</type>

3) And I use the oem parameters:

   <oem-systemsize>800</oem-systemsize>

   That means 800MB for the system image, swap according to the
   RAM space of the system and the rest for home. I'm pretty
   sure everybody has different requirements at that point   

4) I now create the image of type oem and simply dump (via dd)
   the .raw file on my stick. The sticked is plugged in to my
   test system and if everything works as expected I go to the
   next step:

5) I know the image works so I add the attribute format to my
   oem type.

   <type filesystem="ext3" boot="oemboot/suse-11.1" format="iso">oem</type>

6) When recreating the image I got the install iso with the oem image
   on it. I burn the iso on a CD and give it a test on a test system

7) the installation procedure just does the same as I did before
   manually; it dumps the oem image on the disk. The available disks
   are searched and you need to select one before the process starts.
   If you don't want that you need to take a look in the file

   /usr/share/kiwi/image/oemboot/suse-dump

   I was asked to add at least one question before the system completely
   overwrites the system disk :-)

I hope that helps a bit. Professional documentation about this
topic would really help a lot, unfortunately I'm too busy with
other tasks and also not a good doc writer
