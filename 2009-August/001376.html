Marcus Schäfer wrote:
> Hi,
> 
>> I am building an appliance on SLES 10, using a recent version of kiwi 
>> (3.70) and am specifying that the YaST firewall and network modules 
>> should be included in the appliance:
>>
>> <package name="yast2-network"/>
>> <package name="yast2-firewall"/>
>>
>> As package manager I am using smart
>>
>> <packagemanager>smart</packagemanager>
>>
>> Looking at the log file I can see that the requested packages are installed:
>>
>> 436:Installing yast2-firewall  ######################################## 
>> [ 83%]
>> 493:Installing yast2-network   ######################################## 
>> [ 94%]
>>
>> Yet in the unpacked image the files somehow disappear as
>>
>> UNPACKED-IMG_DIR/usr/shareYaST2/clients
>>
>> is empty.
>>
>> There does not appear to be a message in the log file that alerts me to 
>> the removal of these files/packages.
>>
>> I wonder how this stuff disappears?
>> What do I need to do to get the packages and their content to stick, 
>> i.e. be part of the appliance?
> 
> Well the only reason I could imagine is some config.sh script
> which removes it after the installation is done. I suggest
> to check config.sh first

Thanks, and sure enough the config.sh for the SLE 10 template on which 
this appliance is based has the following:

#======================================
# Remove unneeded packages
#--------------------------------------
rpm -qa | grep yast | xargs rpm -e --nodeps


I suppose this is in config.sh for SLE10-JeOS as YaST should not be part 
of "Just Enough OS".
