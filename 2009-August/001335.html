Hi,

> > The storage repartitioning according to the real geometry and
> > the possible oem- variables in config.xml happens on first boot
> > of the system.
> >
> 
> So as long as I meet the minimum space requirements and the correct drivers are present, this shouldn't be a problem, correct?

yes, and if you don't meet the space requirements the re-partitioning is
skipped and the system boots with the virtual geometry inside the image

> > the lan setup is not part of the kiwi boot process. That can
> > be automated by using yast2 firstboot. You can store the yast2
> > firstboot xml file as part of your image description and kiwi will
> > setup the environment in a way that yast is called with that firstboot
> > file on firstboot 
> >
> 
> Is the lan hardware detected on first boot?

only when using the pxe image type the lan setup is done
inside the initrd. Loading the correct nic driver is done
by udev in a very friendly way so there is no modprobe alias
or something required. The nic setup of course must be done
by some settings given before (yast firstboot xml file)
if you know the image will be used inside a network with
dhcp you also can include a static ifcfg- file telling
the system to get the IP from the dhcp. In this very simple
case you don't need a custom nic setup with yast
