Hi,

> Thanks for your help. I have updated the package, and now its working
> correctly. But right now, the size of the image after the build is
> about 1.1 GB. Manually using the package delete is likely to be a
> difficult task. Any idea how I can get rid of the lib packages
> automatically?

sorry that's hard to answer. As you know the packagemanager takes
care for the dependencies it's always a risk to delete OS data
directly. Basically your image will become much smaller if
you change the attributes patternType and patternPackageType
as follows:

   patternType="onlyRequired" patternPackageType="onlyRequired"

This will install only the required packages but for a desktop
like Gnome or KDE much more is "required" to make that system
really work.

I'm sorry but it looks like you need to check with rpm -qa
if you find packages you don't need and remove them manually
Maybe there is also somebody on the list who knows Gnome much
better than I do and can tell you what the minimum set of
packages is to run Gnome in a way you like it

A lot of space can be freed if you remove the docs in
/usr/share/doc. That's data you could remove without loosing
the system integrity

I agree to make a desktop image small is some work
