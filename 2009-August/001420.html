Using the following combination of oem- elements I managed to break the 
oem image for deployment on a USB stick.

 <!-- Do no create an extra partition -->
        <oem-home>false</oem-home>
        <oem-swap>true</oem-swap>
        <!-- 128 MB swap partition -->
        <oem-swapsize>128</oem-swapsize>
        <!-- 7.5 GB root partition -->
        <oem-systemsize>7680</oem-systemsize>
        <oem-kiwi-initrd>true</oem-kiwi-initrd>

This is reproducible with the live-stick example (config file attached).

I am working with an 8GB stick and like to leave 512 MB to add a FAT 
partition on firstboot. Using fdisk on a system after an attempt to boot 
of the stick produces the following information:

Disk /dev/sde: 8032 MB, 8032092160 bytes
255 heads, 63 sectors/track, 976 cylinders
Units = cylinders of 16065 * 512 = 8225280 bytes
Disk identifier: 0x8ef2c335

   Device Boot      Start         End      Blocks   Id  System
/dev/sde1   *           1           2       16033+   2  XENIX root

The error console when booting from the stick produces something to the 
effect that the file system could not be mounted.

It appears that something gets mixed up. I tried to take pictures of the 
screen (attached).

My goal is to have a swap partition on the stick, and some space left 
over to allow the creation of a VFAT partition when teh system boots of 
the stick.

Help is appreciated. Created on openSUSE 11.1.
