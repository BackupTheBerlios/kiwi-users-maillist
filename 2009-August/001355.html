> > Hi,
> > 
> > with the gfxboot-branding and bootsplash-branding packages
> > SuSE provides the possibility to have custom brandings. But
> > many people just don't want to create a package from their
> > data. According to this kiwi now allows you to pack your
> > branding data into a tarball and include that which makes
> > your branding tasks a bit easier.
> > 
> > Well you still need to know where to place the branded
> > files inside the archive.
> > 
> > Example:
> > * The following example shows how to use my custom grub boot
> >   screen (see attachment *don't laugh I'm a bad designer* :)
> >   based on the suse-11.1-JeOS template
> > 
> > 1) cp brand.tgz /usr/share/kiwi/image/suse-11.1-JeOS
> > 2) edit /usr/share/kiwi/image/suse-11.1-JeOS/config.xml
> >    * remove standard branding gfxboot-branding-openSUSE
> >    * add my branding requires gfxboot-devel
> > 
> >      <archive name="brand.tgz" bootinclude="true"/>
> >      <package name="gfxboot-devel" bootinclude="true" bootdelete="true"/>
> > 
> > That's it, build the JeOS and see my great grub boot background :-))
> > All this with kiwi >= v3.60
> 
> either that way or you can create your own branding package as
> they exist in the distribution:
> 
>    gfxboot-branding-...     => graphics boot screen (grub and isolinux)
>    bootsplash-branding-...  => bootsplash background on boot
> 
> Please note if you use a simple tarball with branding data you
> have to take care that the files you store are at the correct place
> according to the used gfxboot version. The example I made works for
> the new style gfxboot which we have with suse 11.0 and later but
> for older distro versions the structure of the gfxboot
> and bootsplash data is different
