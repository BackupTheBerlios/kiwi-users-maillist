Hi,

> I'm probably missing something simple here.  When I create a
> disk image(oem) which can then be copied to a hdd, at what point
> in the process is the actual storage/lan hardware in the physical
> system detected and configured.

The storage repartitioning according to the real geometry and
the possible oem- variables in config.xml happens on first boot
of the system.

the lan setup is not part of the kiwi boot process. That can
be automated by using yast2 firstboot. You can store the yast2
firstboot xml file as part of your image description and kiwi will
setup the environment in a way that yast is called with that firstboot
file on firstboot 

> Also, what drivers are available for this configuration?

the available drivers are part of the boot image descriptions
provided by kiwi in /usr/share/kiwi/image/oemboot/suse-...
normally most of the relevant storage drivers are available.
If there are drivers missing they can be added. kiwi builds
the initrd according to that boot image descriptions
