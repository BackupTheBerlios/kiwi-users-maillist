For an OEM image based on SLES 10 with a recent version of Kiwi 
(kiwi-3.74-44.1) not accepting the EULA does not cause the system to 
shutdown, rather the state off the machine is returned to the EULA 
display in what appears to be an endless loop. Looks like the "No" 
condition for the EULA is not handled correctly.

If the user should get a second or third chance to accept the license it 
appears that a counter is needed in KIWILinuxRC.sh.  However, I think if 
the user does not accept the EULA the system should just shutdown. I 
think the following should do the trick.

    case $? in
            0 ) break
                ;;
            1 ) systemException \
                    "License not accepted... reboot" \
                "reboot"
                ;;
        esac

Not sure if we should turn the system off, i.e. "shutdown -h" or simply 
reboot as in the code above.
