> The original seed of the root tree was based on the JeOS template
> installed on the 32 bit machine.
> 
> I wonder if there were any harm to add the lib64 path to the xorg.conf
> file by default for all examples/templates. This would avoid this error
> for users.

fixed...

> This however makes we wonder about architectures in a more general
> sense. What if I were to run kiwi on Itanium with the same config file
> and directory layout. I guess my question is how does one create a
> platform independent configuration and root tree layout?

- in terms of packages the package manager will pick up the right
  packages if the repositories provides it automatically

- in terms of configuration files this is an individual task. On
  a 64bit system there is lib64 and if the application needs to
  distinguish this it could be intelligent like Xorg on SLE11 which
  doesn't need a modulepath setup or it could be dumb and requires
  a static path list which it searches for the right contents in
  like Xorg on SLE10 does. In other words it's up to to the one who
  creates the image description how well it is prepared for multiarch
  configurations
