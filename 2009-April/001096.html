Hi,

> Is there a macro/function/variable that can be queried by the user in 
> config.sh or image.sh that provides the architecture? Kiwi knows how to 
> figure this out and does, therefore the user, i.e. person setting up the 
> image shouldn't have to invent their own tricks to figure this out if there 
> were some platform specific code.

you can do the following in your script:

case $(uname -m) in
    i686)   echo "32bit x86" ;;
    x86_64) echo "64bit x86" ;;
    *)      echo "arch not handled" ;;
esac

I think I don't need to provide a function for "uname -m" that
looks short enough :)

> Constructing a use case for this is fairly simple. If I am an ISV and want 
> to build an appliance for x86, x86_64, itanium, and ppc, I would build my 
> proprietary binaries on the respective machine and then store them in some 
> directory on a file server. In config.sh based on the platform I am running 
> on I will then copy my architecture specific binaries from the fie server 
> into the unpacked image. With this the configuration is platform 
> independent at a high level and the platform dependency is encapsulated in 
> config.sh

I agree that scenario is a real life example
