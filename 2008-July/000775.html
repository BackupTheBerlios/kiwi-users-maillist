Hi,

> I am building an image for a diskless system. As I mentioned way back on
> the list, I have issues with nbd when booting over a fast network. The
> nbd developers want me to replace the nbd-client with a newer version
> (2.9.11). I have built the new version as binaries and would like to use
> them. My first attempt was to replace the old nbd-client with my new one
> when my install.sh script runs. In fact, this works. But I have a
> feeling that the nbd-client that gets replaced at this time is not the
> one used when mounting / via nbd during boot. It is the one I see after
> the mount is finished. That is, the nbd-client that would get replaced
> when the install.sh script gets run is not the one in the initrd image,
> but the one in the image that nbd eventually mounts.
> 
> Could it be this way? And if so, what is a better way to be sure that my
> nbd-client is the one used during initial boot?
> 
> Sorry if this is not a clear description...

you need the new version of nbd-client in your boot image (initrd)
that's correct. In order to have it there you can copy the binary
to the correct boot image description.

Example:

  boot image is /usr/share/kiwi/image/netboot/suse-11.0

In this case the following should work:

	cd /usr/share/kiwi/image/netboot/suse-11.0
	mkdir -p root/usr/sbin/
	cp nbd-client root/usr/sbin/

re-create your image.

NOTE:
you now have a binary file in your boot image description.
This is evil :) but serves best to give it a test with a new
version of a program. If you know the new version works you
should have the program as part of a package and let kiwi install
it
