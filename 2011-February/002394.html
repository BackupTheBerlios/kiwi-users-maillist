Hi,

> How can I get the script to use the name it discovered and used when
> writing the /etc/fstab, when it is adding the partition and formatting
> it? Or is there some other solution?

kiwi stores the disk device name in the variable 

  $imageDiskDevice

it should be possible to access the value from within your
script if the kiwi initrd called init(). That's because the
exported environment should be available to all sub-processes
I did not test this though

Other than that you can find the matching root device by
parsing proc/mounts. The disk-by-id values can be resolved
into raw device names and so you can get the disk.
