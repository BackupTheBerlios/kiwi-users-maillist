Sorry I have no clue what you are doing. Here is a short howto
to get started:

1) make sure your kiwi installation is not rotten.
   The following uninstalls all kiwi packages
   
   for i in $(rpm -qa | grep kiwi);do rpm -e --nodeps $i;done

2) now install 4.73

   kiwi
   kiwi-tools
   kiwi-desc-vmxboot
   kiwi-desc-oemboot
   kiwi-templates

3) now build an oem JeOS image. I guess you are using 11.2

   mkdir /tmp/myimage
   kiwi --build suse-11.2-JeOS --type oem -d /tmp/myimage

4) now test the image in for example vmware, kvm, virtualbox
   a real disk... whatever you prefer. Following is an example
   for a test with kvm

   qemu-img create mydisk 5G
   qemu-kvm -cdrom LimeJeOS-openSUSE-11.2.i686-1.11.2.iso -hda mydisk

You will be asked to install the system on the disk. After
that operation is done the new system on the disk will be resized
according to the oem setup. The default setup in the 11.2 JeOS
will create a 2G root partition and some swap (depending on the
amount of RAM). After that the system boots up and that's it 
I'm sorry I don't see any of the mentioned problems you had.

> The JeOS seem to be Xen/VMX. I am getting confused.

The JeOS is just a small image description and we make them in
a way that you can build the following image types out of it

 * Live CD
 * Virtual Disk (vmx)
 * Virtual Disk OEM, installs on disk (oem)
 * Virtual Disk Xen guest domU (vmx with profile xenFlavour)
 * Virtual Disk OEM dom0, installs on disk (oem with profile xenFlavour) 

If you don't specify any profile and type a Live CD is build.
But as you are talking about oem (and I think Xen is not of interest)
 added '--type oem' in the howto above. I think you just built
a live cd and was wondering why it did not behave like an oem ?
just guessing

I'm a bit confused what your goals are. Anyway I hope it
helps what I wrote and didn't confuse you even more :)
