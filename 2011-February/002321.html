Hi,

> how can I configure the image, so that it can get the users from the
> NIS/YP? I see, that you have <users/> element, but it does not seems
> to me to be able to handle that.
> 
> Should I simply add +:::: lines to the config files (/etc/passwd and
> so on) with post-prcoessing scripts? Which one should I use for
> that: images.sh or config.sh?

yep just add the nis escape sequences in the passwd and group
files by some script code in config.sh. Additionally you need
to setup /etc/sysconfig/ypbind and make sure you have inserted
the ypbind service on boot.

If you have yast installed in your appliance you also can
use the nis commandline interface in your config.sh script:

   yast2 nis interactive < input

where input is a file with instructions like this

   configure server=... domain="..." automounter=[yes|no]

yast write all required configurations out to the system.
You still need to make sure that the ypbind service is active


Totally untested:

   echo "configure server=192.168.1.1 domain="foo" automounter=no" > input
   yast2 nis interactive < input
   suseInsertService ypbind
