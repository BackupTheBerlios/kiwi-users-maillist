On 02/21/2011 11:50 AM, Marcus Schäfer wrote:
> Hi,
> 
>> I am making an OEM preload image. I set the size of the root partition
>> and of the swap space. This works great. I want to fill the rest of the
>> disk with /home. As I understand it, the oem preload no longer does
>> this. But it used to I guess as the docs all claim this is what it does.
>> But not any more.
> 
> The docs has been fixed here
> 
>> So, I am considering alternatives. One suggestion was to use the
>> <systemdisk/> method and get a logical volume. This is not really what I
>> am after. I want good old fashioned disk partitions: one for / and one
>> for /home.
> 
> I'm sorry but kiwi supports custom partitions only with LVM. The LVM
> overhead is small and you gain persistent device names and also a lot
> more power over your partitions compared to the old model. And as soon
> as the disks start to look over the 2TB border other problems will 
> occur. LVM is used in enterprise environments and it's stable. So
> I would say give it a try
> 
>> I am using yast2-firstboot. Is there any way to tell it to add a
>> partition that fills the rest of the disk? I looked through the docs and
>> did not recognize such a thing. I am not adding any user in the preload,
>> so home is empty. Adding users will happen after the system is set up.
> 
> yast2 storage and also tools like gparted can help you here. Filling
> the rest of the disk can also be done very easily with fdisk in a script
> 
>> If yast2-firstboot does not work, could I use Autoyast? I have the
>> feeling you should do either yast2-firstboot or Autoyast, but not both.
>> For my own sanity, I would like to focus on one method. But I am
>> flexible.
> 
> Autoyast also works. You can call the storage module with some predefined
> input telling it to add a partition filling the rest of the disk. That
> shouldn't be that hard. You could contact Uwe Gansert for help here
> <ug at novell.com>
> 

Just to add to the info Marcus already provided. You can also follow the
data separation example:
http://en.opensuse.org/SDB:KIWI_Cookbook_Data_Separation to create the
extra partition. Then in a script you can reset /etc/fstab
