> Well, I switched back to kiwi-4.43-1.6, which is the default in OpenSuSE

Don't use the default from 11.3 use the 4.43 version from the home:ctso
repo I posted. We found and fixed several problems after 11.3 GA

  http://download.opensuse.org/repositories/home:/ctso/openSUSE_11.3/
   
> Any clue here? is there something with the parted inside SLES10 that this
> version of kiwi doesn't know how to handle?

yes parted from sle10 is too old it doesn't support the "-m" option.
iirc we added a check in kiwi to switch to fdisk in that case. You also
might consider to update parted and use a newer version for sles10

grmbl, sles10 imaging is really not an easy task, zypper too old
parted too old and btw you always have to pass the following option
to kiwi when building a sles10 image on a 11.3 build system

   --fs-inodesize 128
