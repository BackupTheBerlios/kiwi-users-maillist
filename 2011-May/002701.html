All,

Please be aware that our documentation for the <preferences> element is 
currently misleading. :( sorry about this.

I'd like to illustrate this using the <size> element as an example. At 
the beginning of section 2.1.4 the <size> element is shown to be a child 
of <type>. Yet, as all child elements of the <preferences> element are 
discussed in section 2.1.4 the distinction between child elements of 
<type> and chile elements of <preferences> is somewhat lost leading to 
confusion whether <size> should be a child of <preferences> or <type>.

Ordinarily this would not necessarily be all that big of a deal. 
However, the problem is that there exists a parser bug (at least for 
libxml2 and the Perl interface on openSUSE 11.3 and most likely openSUSE 
11.2, and SLES 11 SP1) that passes a config.xml file that has <size> 
specified as a child of <preferences> as valid :(

Just thought everyone should be aware of this. Other child elements of 
<type> may be effected by this parser bug. A way to verify you XML 
outside the Kiwi process is to install jing and use as follows:

-> jing /usr/share/kiwi/modules/KIWISchema.rng PATH_TO_YOUR_CONFIG.XML_FILE

jing catches the previously mentioned structural error for the <size> 
element.
