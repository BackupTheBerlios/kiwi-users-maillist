We have a funny piece of hardware that we want to boot with KIWI/PXE.
The problem is that it has two NIC, but only one of them allows booting
via PXE. The initial part seems to be ok. But at some point, the kiwi
boot process does a dhcp request on eth0. It is at the point shortly
after the boot process prints "Loading KIWI PXE Boot-System..." At that
time, a DHCP request ss done on eth0.

As it turns out, this is not the port we can enable to do a PXE boot. We
have looked to see if we can disable this NIC in the BIOS. But either it
cannot be done, or it is very well hidden.

My question is, is there a way to specify, at boot time, which NIC to
use for this dhcp request?
