It looks like 4.89 has an issue when building with bootincluded archive files.  The prepare seems to be ok, but the build fails with an error saying it "Can't find" the archive file in the image-root directory that kiwi is currently using.
This can quickly be reproduced by adding a license.txt file (tarred and then gzipped) in a license.tar.gz to the suse-SLE11-JeOS sample directory and adding a kiwi hook script in a kiwi-hooks.tgz, then adding the following lines top of the packages list in the config.xml:
                <archive name="EMCLicense.tar.gz" bootinclude="true"/>
                <archive name="kiwi-hooks.tgz" bootinclude="true"/>
Building the suse-SLE11-JeOS sample will now fail with the following error:
May-20 15:00:38 <3> : Can't find /tmp/kiwitmp/build/image-root/EMCLicense.tgz
May-20 15:00:38 <3> : Archive installation failed   failed

For now I think we can work around this by copying the archives into the prepared image-root directory manually in between the prepare and build steps.
