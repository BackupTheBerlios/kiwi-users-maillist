How is kiwi dealing with the LVM uuids for Physical Volumes, Volume Groups, and Logical Volumes?
It seems that every system installed with the same version of a kiwi generated  OEM install type cd has the exact same uuids for all the LVM objects.

This is easily seen if you use an OEM type install(with LVM partitioning) on two systems, and compare the output of pvdisplay/vgdisplay/lvdisplay on the two systems.  The objects that were dd'd onto the system by kiwi have the same uuids.  Ones that were generated during the system install have different uuids.

Shouldn't a new install always give new uuids to the installed partitions?
Is there are way to have the kiwi OEM installs update these lvm objects with unique uuids during the installation?
