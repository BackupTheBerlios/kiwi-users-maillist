Hi Justin,

> I've got some basic newbie questions concerning the use of kiwi to build 
> netboot/pxeboot images.
> 
> What is the difference?

the netboot term references the boot image (the initrd) which controls
the boot process initiated by a PXE boot based protocol. Along with
this boot image there is also a package called kiwi-pxeboot. This packages
provides the basic directory structure and some PXE default configuration
files which are sufficient to work with the netboot boot image. 

> In the documentation, there is a reference to the 
> ability to use kiwi to create a bootable image that will be held 
> completely within the client's memory.  But I'm also getting the 
> impression that one is intended for use in a case where the client would 
> be installing based on the kiwi image.

yes both is possible and even more. The netboot boot image is able
to work in different modes:

- system image completely in RAM
- system image partially in RAM and local storage
- system image completely on local storage
- system image remotely mounted via NFS (network file system)
- system image remotely mounted via NBD (network block device)

> My goal is to serve an image (via NFS or NBD, it doesn't make a difference 
> to me) that completely resides in memory. I want to be able to reboot the 
> client at any point and just be back to the original image.  The only 
> application the client will run is rdesktop, so I don't need audio or any 
> local storage whatsoever.
> 
> I'm currently using kiwi-ltsp to accomplish this, but I really don't think 
> that I need the terminal server aspect that this features.  I just want 
> the client to essentially load a live-image and run rdesktop in full 
> screen mode.

I agree you don't need the terminal aspects here. So to make this work
you need the following:

1) a system image tree exported via NFS or NBD
2) a PXE boot server based on the directory structure from the
   kiwi-pxeboot package
3) a netboot boot image and kernel built with kiwi
4) a config.<MAC> configuration file which includes one line
   telling netboot to use NBD or NFS

Would you mind to jump in and tell us according to the list
above at which point you are currently and what problems needs
to be solved.
