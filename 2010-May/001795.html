Changes Report v4.34 2010-05-02 - 2010-05-07
--------------------------------------------------
I apologize for any regressions introduced with the last kiwi
version and would like to thank you all for reporting bugs that
quickly. I hope everything runs smooth now with this weeks
release

most important changes for this weeks kiwi release...

GIT: fixed old live iso type (no flags set) to work again.
     The problem was that we prevent --follow-symlinks in this
     mode but then we also have to copy the fixed read-write
     part as file to the iso structure instead of linking it.
     This fix is related to (bnc #599712)

* If you use an old style iso specified like this
  <type image="iso" boot="..."/> without any flags attribute
  kiwi failed to boot this image because of a link which should
  be followed but won't be since we changed that due to bug
  #599712. With this fix the old style iso's works again


GIT: removed check for utimer program, it's not a mandatory
     requirement for the system to be able to boot correctly
GIT: fixed mkinitrd call, check if the option -B exists and
     don't pass it if it doesn't first (bnc #603338)

* This is the most important fix for this weeks release. Every
  image for distros <= 11.1 will fail the second boot because of the
  missing -B option for mkinitrd. Another reason for a failed
  boot could also be the missing utimer helper program which
  is part of the kiwi-tools package. If this program is missing
  you will not see the boot time values but this causes kiwi
  to throw a reboot exception which is stupid and not necessary


GIT: beautify migration report and fixed html code

* The migration module now has some css code to make the HTML
  report look nicer. The HTML code is now also valid


GIT: fixed space problem when setting up boot partition for
     images which requires a separate boot partition which is
     the case for filesystems like clicfs. This caused a wrong
     copy of the initrd and prevents the system from being
     able to reboot

* also an important fix for all oem users. In case of a separate
  boot partition the size of this partition was too small. The
  result was that the initrd can't be copied to this partition and
  a reboot of the appliance failed with a kernel trace because
  of an invalid initrd cpio archive

GIT: added support for Xen dom0 over netboot (bnc #601890)

* I added a xen + xenk kernel profile to the netboot images to
  be able to create pxe images with Xen kernel and appropriate
  boot setup. If you build such an image your type line should
  look like this: <type image="pxe" bootprofile="xen" bootkernel="xenk" .../>
  Along with this change the netboot code uses now the common
  partition functions like all other images too.
