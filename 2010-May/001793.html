Changes Report v4.32 2010-04-25 - 2010-04-30
--------------------------------------------------
Summary of the most important kiwi changes in master...

GIT: fixed createHybridPersistent: wait for new storage device
     before creating a filesystem. Also make sure imageDiskDevice
     is set before calling callPartitioner()

  The hybrid live system support with persistent write partition
  support was broken after the change from fdisk to parted. parted
  does not accept the partition table written into the ISO as a
  valid partition table. Therefore I had to temporarly switch back
  to fdisk for all hybrid related partition operations. Along with
  this change it was also important to wait for the new storage
  device after repart and also set the imageDiskDevice variable
  appropriately

GIT: fixed repartition code for overlay systems (clicfs)

  When using the image type oem together with filesystem="clicfs"
  the repartition code to the real geometry with parted was broken
  Additionally the information for reboot (kernel + kiwi initrd)
  such a system was missing too


GIT: support new createrepo repomd.xml metadata information.
     kiwi now reads the repomd.xml if it exists and parses the
     primary data section in order to get all metadata file names

  This is an important fix for users who use the opensuse build
  service repositories for building images. The BS supports the
  new createrepo format which now makes it necessary to read the
  repomd.xml file if it exists in the repodata/ directory. This
  is required in order to find the correct file name which stores
  the metadata for this repo. It was also required to implement
  a timestamp check to check if it's required to update the cached
  repodata information
