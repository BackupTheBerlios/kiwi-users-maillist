Changes Report kiwi v4.37 2010-05-16 - 2010-05-21
--------------------------------------------------
GIT:  fixes endless building problem if function
      suseInsertService() is calling in config.sh and system/chroot
      contains a broken /etc/init.d/ script. (bnc #605981)

  Uwe Gansert fixed this potential loop problem when using
  the suseInsertService() function


GIT:  removed support for dmsquash filesystem. This implementation
      was replaced by clicfs. dmsquash was a combination of a
      squashfs filesystem together with a device mapper snapshot
      as copy-on-write part. There was no way to properly react
      if we run out of cow space which also makes this feature
      more or less useles in a production environment

  *** git comment explains this good enough imho ***

GIT:  rewrote the oem partition code for better maintenance
      and to support syslinux/extlinux bootloader (bnc #606651)

  The hardest part of this weeks release was the required
  restructuring of the oem image re-partition code. We want to
  support extlinux for the MeeGo project in the future and this
  was the initital trigger to make that code much better. I hope
  I did not introduce any regressions. If you find one please
  do not hesitate to report back the issues
    

GIT:  added support for type specific package sections
      <packages type="iso|net|usb|oem|xen|vmx">. Along with this
      change a new Schema version 4.6 was introduced which
      automatically converts the type="vmware" section into
      a type="vmx" section

  This allows you to have a packages section like the following

  <packages type="vmx">
     <package name="..."/>
  </packages>

  this section applies only if you build the image type 'vmx'

GIT:  added support for extlinux bootloader for vmx and usbboot.
      support for extlinux and oemboot will follow later. The
      repartition code of the oem images causes a more complex
      implementation for sys|extlinux.

  This implementation is not completely finished but we have
  a first version which allows to use extlinux for vmx and usb.
  In order to activate it just set <type ... bootloader="extlinux"/>

Have fun :)
