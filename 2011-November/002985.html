Hello there!

I read through some of the archive, but could not find a solution to my 
problem on this mailing list neither. What I want to do, is create a 
simple PXE boot client with KIWI. I've done the preparations following 
the manual, and successfully created the image. I also set up the 
server, to serve the image and run the services neccessary, but the ram 
only client gives the following error:

/sbin/init no such file of not executable

Now I've been looking for a solution for near a week, but I did not find 
any solution that works for me.

The following briefly describes the method I am using now:

1. Prepare the image
kiwi -p /usr/share/doc/packages/kiwi/examples/suse-11.4/suse-pxe-client 
--root /data/kiwiprepare

2. Modify the configuration in /data/kiwiprepare/image/config.xml

<type image="pxe" filesystem="ext3" boot="netboot/suse-11.4">
</type>

3. Create the image
kiwi -c /data/kiwiprepare -d /data/kiwicreate

4. Copy images, initr and create configuration files

/srv/tftpboot/KIWI/config.default:

IMAGE=/dev/ram1;suse-11.4-pxe-client.i686;1.2.8;10.0.0.1;4096

I do not really understand, why KIWI tests /mnt/sbin/init as it is a 
symlink to /bin/systemd this it will not work if the image is mounted on 
/mnt.
I probably miss something in my configuration (in config.xml), but I 
could not find any help, how to configure a ram only PXE boot client.
