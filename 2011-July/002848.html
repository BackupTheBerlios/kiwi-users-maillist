As I said this is not possible. The architecture does not match
If you try to install for example the joe editor 64bit on a 32bit
machine you get:

rpm -Uhv --nodeps /mnt/suse/x86_64/joe-3.7-12.3.x86_64.rpm 
Preparing...                ########################################### [100%]
        package joe-3.7-12.3.x86_64 is intended for a x86_64 architecture

the --target-arch option will not change this. You can use target-arch
only within a limited space. It will for example allow you to install
all i686 optimized packages on a x86 machine, or force all 32bit packages
on a 64bit machine. But it cannot overcome the architecture specific
design. Only a virtual machine emulating the architecture would be
able to do this
