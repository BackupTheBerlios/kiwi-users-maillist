Hi Roger,

> > Based on some docs elsewhere (http://desire.sk/simple/node/5), I think I
> > should copy the file fsdata.ext3 elsewhere and make the change.
> > 
> > 	mkdir -p cmnt
> > 
> > 	clicfs my.clicfs cmnt
> > 
> > 	resize2fs cmnt/fsdata.ext3
> > 
> > However, when I try to copy it, I get this error:
> > 
> > 	cp cmnt/fsdata.ext3 .
> > 	cp: reading `cmnt/fsdata.ext3': Bad address
> > 
> > I got the same error if I did the resize2fs or not. It copied 99% of the
> > file when the error occurred. No message in /var/log/messages.
> > 
> > I am doing this on an 11.2 system with clicfs-1.3.10-49.1, which is the
> > same as was used when the image was first made with kiwi.
> 
> Oddly, I can mount the file system and copy everything from inside the
> ext3 image with cp -a. But I cannot copy the ext3 image itself from the
> clicfs without getting the previously reported error.

you have to give the clicfs at least some bytes of memory

  clicfs -m 512 my.clicfs cmnt
  cp cmnt/fsdata.ext3 .
 
> So, I have copied the contents of the ext3 file system to a directory
> and made my changes. Now I need to make an ext3 file system from this
> directory. Just like kiwi did. Any pointers?

Just take a look into /usr/share/kiwi/modules/KIWIImage.pm
and check the code in the createImageClicFS() function. You
need to repeat the steps there in order to create a new clicfs
filesystem
