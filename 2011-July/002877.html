Hi,

> I am not sure that the new features really preclude the current
> feature, i.e. why would the --prebuiltbootimage option need to be
> removed. I have not looked at the code, but it cannot be all that
> much code or all that difficult to maintain.

Well it's difficult to track down problems related to this

> My thought is that this is still very useful for an oem image where
> the hardware is very well defined and the initrd can be cut down and
> targeted for just the given hardware.

Today you keep the initrd and if we do the change you will
keep the unpacked tree from which the initrd is created. 
Or you decide to use the new image cache system

> I am not opposed to the idea, just not sure why we would need to
> remove the existing option in this case.

I don't like to deal with the potential risk of that feature.
I think it requests a lot knowledge from our users about the
internal processes running in kiwi when dealing with the boot
image. That's not user friendly and I think kiwi should just
do the right thing. Even me stumbles over that from time to time
when I forgot that I have added something in my system image which
requires a rebuild of the boot image, but the system insists to
use the static prebuilt version

That's why I think it should go away

Marcus
