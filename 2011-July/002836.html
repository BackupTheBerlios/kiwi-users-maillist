The permissions module of the SuSEConfig tool also checks
/usr/lib/hal/hald-generate-fdi-cache. But it seems that this
file does not exist in your image. The cache file belongs to the
hal package. So you should check why it's not there:

1) is hal installed
2) did your scripts remove something

kiwi treat the config.sh call as failed if you leave the script with
an exit code != 0. You can ignore all kinds of errors if you have an
'exit 0' at the end of your script and kiwi will continue

But be careful it's not always wise to ignore errrors :)
