On Tue, 2011-07-26 at 13:16 +0200, Roger Oberholtzer wrote:
> I would like to change a file in a clicfs file system. It is the root of
> a diskless kiwi. I can mount and the file system and edit the file. 
> 
> 	mkdir -p cmnt fmnt
> 
> 	clicfs -m 512 my.clicfs cmnt
> 
> 	resize2fs cmnt/fsdata.ext3
> 
> 	fsck  -f -v `pwd`/cmnt/fsdata.ext3
> 
> 	mount -o loop cmnt/fsdata.ext3 fmnt
> 
> But I am unsure how to save it. I thought if I unmounted the file
> system, the changes would be present. Seems not. Any pointers? I suspect
> it is related to the -m option to clicfs. 
> 
> Perhaps I need to do this another way? Copy the contents somewhere in
> the regular file system, make the change, and remake the clicfs from
> that?

Based on some docs elsewhere (http://desire.sk/simple/node/5), I think I
should copy the file fsdata.ext3 elsewhere and make the change.

	mkdir -p cmnt

	clicfs my.clicfs cmnt

	resize2fs cmnt/fsdata.ext3

However, when I try to copy it, I get this error:

	cp cmnt/fsdata.ext3 .
	cp: reading `cmnt/fsdata.ext3': Bad address

I got the same error if I did the resize2fs or not. It copied 99% of the
file when the error occurred. No message in /var/log/messages.

I am doing this on an 11.2 system with clicfs-1.3.10-49.1, which is the
same as was used when the image was first made with kiwi.
