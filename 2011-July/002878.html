On 07/27/2011 11:26 AM, Marcus Schäfer wrote:
> Hi,
>
>> I am not sure that the new features really preclude the current
>> feature, i.e. why would the --prebuiltbootimage option need to be
>> removed. I have not looked at the code, but it cannot be all that
>> much code or all that difficult to maintain.
>
> Well it's difficult to track down problems related to this

Yes, it is difficult to track problems with the initrd. However, this 
difficulty does not go away with the removal of the current option. For 
the most common case, i.e. let kiwi create the initrd nothing changes 
either.

>
>> My thought is that this is still very useful for an oem image where
>> the hardware is very well defined and the initrd can be cut down and
>> targeted for just the given hardware.
>
> Today you keep the initrd and if we do the change you will
> keep the unpacked tree from which the initrd is created.
> Or you decide to use the new image cache system

Well, the basic underlying assumption with this is that kiwi created the 
initrd. What if I do not use kiwi to create the initrd?

I am making an argument for a use case that is outside of "mainstream" 
use of kiwi, that is correct. But we have other features that fit that 
description.

>
>> I am not opposed to the idea, just not sure why we would need to
>> remove the existing option in this case.
>
> I don't like to deal with the potential risk of that feature.
> I think it requests a lot knowledge from our users about the
> internal processes running in kiwi when dealing with the boot
> image. That's not user friendly

Well, user friendliness is in the eye of the beholder. If I cannot use 
my own initrd anymore that's not necessarily user friendly either ;).

Don't get me wrong, I am not married to this feature, and I am with you 
in not wanting to spend time debugging other people's initrd problems. 
However, there are valid use cases for this feature as an "advanced user 
feature". Speak, use it and you're on your own.

> and I think kiwi should just
> do the right thing.

Yes, kiwi should just do the right thing, we need to work on that ;) but 
that's another thread.

> Even me stumbles over that from time to time
> when I forgot that I have added something in my system image which
> requires a rebuild of the boot image, but the system insists to
> use the static prebuilt version
>
> That's why I think it should go away

I like the feature as a "use at your own risk" feature, but if no one 
else on this list chimes in and really cares to keep the feature than 
yes, lets get rid of it as the feature is apparently not being used.
