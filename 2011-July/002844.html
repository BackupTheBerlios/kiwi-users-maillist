There could be the following reasons:

1) the requested package does not exist as 64bit version in the repo set ?
2) you are building the image on a 32bit machine, check with 'uname -m' ?
3) you are building the image prefixed with 'linux32' tool ?
4) you explicitly set <package name="..." arch="i586"/> in config.xml ?
5) you used the --target-arch="i586" option when building ?

In any case the config.xml and the entire log file is helpful
